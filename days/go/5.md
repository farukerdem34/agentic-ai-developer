# Day 5: Go Fundamentals Review Project

## 📝 Today's Tasks

1.  **Project Goal:** Build a small CLI that greets users and performs a simple calculation or text transform.
2.  **Use Packages:** Split `main` and a helper package with exported functions.
3.  **Handle Input:** Read flags or stdin and validate basic input.
4.  **Format and Build:** Run `go fmt`, `go vet`, and produce a binary with `go build`.

## 📖 Task Explanations

Today consolidates days 1–4. You will combine packages, types, functions, and basic I/O into a tiny but complete CLI. Focus on clarity, idiomatic naming, and a clean module layout rather than features.

## 💡 What You'll Gain

A practical, hands-on understanding of Go fundamentals: modules, packages, types, and building a runnable binary.

## 📚 Dictionary for Developers

*   **CLI:** Command-line interface—a program run from a terminal.
*   **Idiomatic Go:** Code that follows community conventions for clarity and simplicity.
*   **`go vet`:** A static analysis tool that finds common mistakes in Go code.
