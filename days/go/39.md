# Day 39: Migrations

## 📝 Today's Tasks

1.  **Migration Tooling:** Adopt golang-migrate, goose, or atlas for schema migrations.
2.  **Up/Down:** Write an initial schema migration and a reversible change.
3.  **Apply in Dev:** Run migrations as part of local setup.
4.  **Versioning Discipline:** Never edit applied migrations; add new ones.

## 📖 Task Explanations

Migrations make schema changes repeatable across environments. Treat them as code reviewed artifacts.

## 💡 What You'll Gain

Repeatable schema evolution with a migration tool.

## 📚 Dictionary for Developers

*   **Migration:** A versioned change to the database schema.
*   **Schema Drift:** When environments' schemas diverge unexpectedly.
*   **Reversible Migration:** A migration that provides a safe down path.
