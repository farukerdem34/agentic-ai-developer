# Day 2: Packages, Imports, and Project Layout

## 📝 Today's Tasks

1.  **Create Multiple Packages:** Add a `main` package and a small library package (e.g., `greeter`).
2.  **Export vs Unexport:** Practice exported (Capitalized) and unexported identifiers.
3.  **Import Paths:** Import your library package using the module path.
4.  **Build from Root:** Build the binary with `go build` and run the resulting executable.

## 📖 Task Explanations

Go organizes code into packages. The `main` package produces an executable; other packages are libraries. Visibility is controlled by capitalization: names starting with an uppercase letter are exported. Understanding package boundaries early prevents messy designs later and matches how professional Go repos are laid out.

## 💡 What You'll Gain

Today you will organize code into packages, understand export rules, and build a binary from a multi-package module.

## 📚 Dictionary for Developers

*   **Package:** A directory of Go files that share a package name and compile together.
*   **Exported Identifier:** A name starting with an uppercase letter, accessible outside its package.
*   **Import Path:** The module-qualified path used to import another package.
*   **`main`:** The package that defines a program entry point via `func main()`.
