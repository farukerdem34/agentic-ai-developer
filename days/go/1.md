# Day 1: Introduction to Go and Toolchain

## 📝 Today's Tasks

1.  **Install Go:** Install the latest stable Go toolchain from the official site and verify with `go version`.
2.  **Set Up Your Workspace:** Create a working directory and initialize a new module with `go mod init`.
3.  **Write Hello World:** Create a `main.go` that prints a greeting and run it with `go run`.
4.  **Explore go Commands:** Try `go build`, `go fmt`, and `go env` and note what each does.

## 📖 Task Explanations

Go is a statically typed, compiled language designed for simplicity, concurrency, and fast builds. Today is about installing the toolchain, creating your first module, and running a tiny program. Modules (`go.mod`) are how modern Go manages dependencies and package paths. Getting comfortable with `go run`, `go build`, and `go fmt` sets the baseline for every later day.

## 💡 What You'll Gain

By the end of today, you will be able to install Go, create a module, format code, and run a simple program. This is the first step toward writing idiomatic Go services.

## 📚 Dictionary for Developers

*   **Module:** A unit of versioned Go code identified by a module path in `go.mod`.
*   **Toolchain:** The set of Go tools (`go`, compiler, linker, formatter) used to build and manage programs.
*   **`go run`:** A command that compiles and runs a Go program in one step.
*   **`go fmt`:** A tool that formats Go source into the standard style.
