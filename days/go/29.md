# Day 29: Middleware

## 📝 Today's Tasks

1.  **Middleware Signature:** Write middleware as `func(http.Handler) http.Handler`.
2.  **Logging Middleware:** Log method, path, status, and duration.
3.  **Recovery Middleware:** Recover from panics and return 500 without crashing the process.
4.  **Compose:** Chain multiple middleware around your mux.

## 📖 Task Explanations

Middleware wraps handlers for cross-cutting concerns: logging, auth, CORS, recovery. Composition keeps handlers focused on business logic.

## 💡 What You'll Gain

Reusable middleware chains for logging, recovery, and future auth hooks.

## 📚 Dictionary for Developers

*   **Middleware:** A function that wraps a handler to add cross-cutting behavior.
*   **Panic Recovery:** Catching panics in handlers so one bad request does not kill the server.
*   **Cross-Cutting Concern:** Logic that applies across many endpoints.
