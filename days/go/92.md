# Day 92: Module Versioning and Compatibility

## 📝 Today's Tasks

1.  **Semver:** Follow semantic import versioning for Go modules.
2.  **v2+ Paths:** Understand major version suffix rules (`/v2`).
3.  **go.mod Hygiene:** Tidy modules; avoid unnecessary replaces in mainline.
4.  **Dep Updates:** Update a dependency carefully and run tests.

## 📖 Task Explanations

Go's module system encodes compatibility in import paths. Respect semver to avoid breaking consumers.

## 💡 What You'll Gain

Confident module versioning and safer dependency updates.

## 📚 Dictionary for Developers

*   **Semver:** Semantic Versioning—MAJOR.MINOR.PATCH meaning.
*   **Semantic Import Versioning:** Go's rule linking major versions to import paths.
*   **`go mod tidy`:** A command that adds missing and removes unused module requirements.
