# Day 18: Select and Cancellation Patterns

## 📝 Today's Tasks

1.  **Select:** Multiplex multiple channel operations with `select`.
2.  **Default Case:** Use `default` for non-blocking attempts carefully.
3.  **Done Channel:** Propagate cancellation via a done channel.
4.  **Timeouts:** Implement a timeout using `time.After` in a select.

## 📖 Task Explanations

`select` is the switch statement of concurrency. Cancellation and timeouts prevent hung operations. Prefer later `context` patterns, but understand channel-based cancellation first.

## 💡 What You'll Gain

You will multiplex channel ops, implement timeouts, and cancel work cleanly.

## 📚 Dictionary for Developers

*   **Select:** A control structure that waits on multiple channel communications.
*   **Timeout:** Aborting an operation after a deadline.
*   **Cancellation:** Signaling that in-flight work should stop.
