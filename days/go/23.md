# Day 23: Benchmarks and Coverage

## 📝 Today's Tasks

1.  **Benchmarks:** Write `BenchmarkXxx(b *testing.B)` and run with `-bench`.
2.  **Allocs:** Report allocations with `-benchmem`.
3.  **Coverage:** Generate coverage with `go test -cover` / cover profiles.
4.  **Interpret Results:** Compare two implementations carefully (warmup, fairness).

## 📖 Task Explanations

Benchmarks quantify performance; coverage shows untested paths—not a goal by itself. Measure before optimizing, and keep benchmarks honest.

## 💡 What You'll Gain

Ability to benchmark, inspect allocations, and read coverage reports.

## 📚 Dictionary for Developers

*   **Benchmark:** A test measuring performance of a function under looped execution.
*   **Coverage Profile:** A report of which statements were executed by tests.
*   **Allocation:** Heap memory allocated during execution, costly at scale.
