# Day 84: HTTP/2 and Service-to-Service Calls

## 📝 Today's Tasks

1.  **HTTP/2 Benefits:** Multiplexing and header compression at a high level.
2.  **Client Connections:** Reuse `http.Client` / transports; avoid per-request clients.
3.  **Outbound Policies:** Timeouts, retries (careful), and tracing propagation.
4.  **Connection Limits:** Tune `MaxIdleConns` thoughtfully.

## 📖 Task Explanations

Service-to-service traffic needs pooled clients and deadlines. Misconfigured HTTP clients are a classic production footgun.

## 💡 What You'll Gain

Efficient, timed outbound HTTP with connection reuse.

## 📚 Dictionary for Developers

*   **Connection Pooling:** Reusing TCP connections across requests.
*   **Multiplexing:** Multiple streams over one connection (HTTP/2).
*   **Transport:** The `http.RoundTripper` that executes HTTP requests.
