# Day 34: Input Validation Deep Dive

## 📝 Today's Tasks

1.  **Validate Structs:** Validate lengths, formats, and required fields.
2.  **Domain Errors:** Map validation failures to 400 with field details.
3.  **Reusable Validators:** Extract shared validation helpers.
4.  **Security Mindset:** Reject unexpected fields / oversized payloads.

## 📖 Task Explanations

Validation is security and UX. Be explicit about what you accept. Return actionable errors without leaking internals.

## 💡 What You'll Gain

Robust validation layered on JSON APIs with clear client-facing errors.

## 📚 Dictionary for Developers

*   **Allowlist:** Accepting only known fields/values rather than trying to block bad ones.
*   **Payload Limit:** A maximum request body size to prevent abuse.
*   **Field Error:** A validation error tied to a specific input field.
