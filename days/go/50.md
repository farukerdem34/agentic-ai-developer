# Day 50: Auth & Security Practice Project

## 📝 Today's Tasks

1.  **Project Goal:** Add registration, login, JWT (or sessions), and RBAC to your API.
2.  **Protected Routes:** Lock mutating endpoints behind authz checks.
3.  **Secrets:** Load signing keys from config/env.
4.  **Security Pass:** Review password hashing, token expiry, and error messages.

## 📖 Task Explanations

Security is a feature. A complete authN/authZ slice makes your service realistically deployable.

## 💡 What You'll Gain

An API with authentication, authorization, and safer secret handling.

## 📚 Dictionary for Developers

*   **AuthN:** Authentication—verifying identity.
*   **AuthZ:** Authorization—verifying permissions.
*   **Signing Key:** A secret used to sign and verify tokens.
