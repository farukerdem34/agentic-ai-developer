# Day 8: Interfaces

## ЁЯУЭ Today's Tasks

1.  **Define an Interface:** Create a small interface (e.g., `Storage` or `Notifier`) with one or two methods.
2.  **Implicit Satisfaction:** Implement the interface on a concrete type without an `implements` keyword.
3.  **Depend on Interfaces:** Write a function that accepts the interface, not the concrete type.
4.  **Empty Interface Caution:** Avoid overusing `any`; prefer specific interfaces.

## ЁЯУЦ Task Explanations

Interfaces in Go are satisfied implicitlyтАФthis enables loose coupling without inheritance hierarchies. Small interfaces (often one method) are idiomatic. Designing around interfaces makes testing and swapping implementations natural.

## ЁЯТб What You'll Gain

You will design small interfaces, implement them implicitly, and write functions that depend on abstractions.

## ЁЯУЪ Dictionary for Developers

*   **Interface:** A type that defines a method set; any type with those methods satisfies it.
*   **Implicit Satisfaction:** Go's rule that types implement interfaces by having the right methods, with no declaration.
*   **`any`:** Alias for `interface{}`, the empty interface that can hold any value.
*   **Dependency Inversion:** Depending on abstractions (interfaces) rather than concrete types.
