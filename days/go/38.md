# Day 38: Transactions

## 📝 Today's Tasks

1.  **Begin/Commit/Rollback:** Implement a multi-step write in a transaction.
2.  **Defer Rollback Pattern:** Defer rollback and commit only on success.
3.  **Isolation Awareness:** Know basic isolation levels at a high level.
4.  **Handler Integration:** Expose a use-case that requires transactional integrity.

## 📖 Task Explanations

Transactions protect invariants across multiple statements. The defer-rollback pattern is idiomatic and avoids partial writes on errors.

## 💡 What You'll Gain

Correct transactional writes with clean error and rollback handling.

## 📚 Dictionary for Developers

*   **Transaction:** A group of SQL statements that commit or roll back together.
*   **Atomicity:** All-or-nothing application of a transaction.
*   **Isolation Level:** Rules about visibility of concurrent transactions.
