# Day 46: Password Hashing and User Store

## 📝 Today's Tasks

1.  **Hash Passwords:** Use bcrypt or argon2—never store plaintext.
2.  **Register Flow:** Create a user with unique email/username checks.
3.  **Timing/Errors:** Avoid user enumeration where practical.
4.  **Tests:** Cover hash verify success and failure paths.

## 📖 Task Explanations

Authentication starts with safe password storage. Use established KDFs. Keep auth errors carefully worded.

## 💡 What You'll Gain

Secure password registration and verification flows.

## 📚 Dictionary for Developers

*   **KDF:** Key derivation function used to hash passwords slowly and safely.
*   **bcrypt / argon2:** Common password hashing algorithms.
*   **User Enumeration:** Leaking whether an account exists via different responses.
