# Day 64: Detecting Leaks and Deadlocks

## 📝 Today's Tasks

1.  **Common Leak Causes:** Blocked channel sends, missing cancels, forgotten WaitGroups.
2.  **Debug Techniques:** Add timeouts, logs, and `GODEBUG`/`pprof` traces when needed.
3.  **goleak Optional:** Try `uber-go/goleak` in tests for goroutine leak detection.
4.  **Fix Drill:** Introduce and then fix a deliberate leak.

## 📖 Task Explanations

Leaks show up as memory growth and stuck processes. Detect them in tests when possible; always design exit paths.

## 💡 What You'll Gain

Ability to diagnose and fix goroutine leaks and simple deadlocks.

## 📚 Dictionary for Developers

*   **Deadlock:** A state where goroutines wait on each other forever.
*   **Goroutine Leak:** A goroutine that never becomes free to exit.
*   **goleak:** A test helper that asserts no unexpected goroutines remain.
