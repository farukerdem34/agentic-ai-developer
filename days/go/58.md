# Day 58: gRPC Interceptors and Streaming

## 📝 Today's Tasks

1.  **Unary Interceptors:** Add logging/auth interceptors.
2.  **Streaming Intro:** Implement a basic server-stream or client-stream method.
3.  **Context:** Propagate deadlines via gRPC context.
4.  **Keep Deadlines:** Ensure clients set timeouts.

## 📖 Task Explanations

Interceptors are gRPC middleware. Streaming unlocks large payloads and live updates. Deadlines remain non-negotiable.

## 💡 What You'll Gain

Interceptors for cross-cutting concerns and a first streaming RPC.

## 📚 Dictionary for Developers

*   **Interceptor:** gRPC middleware for unary or streaming calls.
*   **Unary RPC:** A single request/response RPC.
*   **Streaming RPC:** An RPC that sends a sequence of messages.
