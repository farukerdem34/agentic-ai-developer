# Day 82: Server-Sent Events (SSE)

## 📝 Today's Tasks

1.  **SSE Handler:** Stream events to a browser/client over HTTP.
2.  **Flushing:** Ensure events are flushed promptly.
3.  **Reconnects:** Understand Last-Event-ID basics.
4.  **When to Use:** Compare SSE vs WebSockets for your use case.

## 📖 Task Explanations

SSE is a simpler one-way realtime channel over HTTP. Great for notifications and live feeds.

## 💡 What You'll Gain

An SSE stream with correct headers and flushing behavior.

## 📚 Dictionary for Developers

*   **SSE:** Server-Sent Events—server-to-client streaming over HTTP.
*   **Last-Event-ID:** A header clients use to resume an event stream.
*   **Flush:** Pushing buffered response bytes to the client immediately.
