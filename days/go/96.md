# Day 96: System Design for Go Services

## 📝 Today's Tasks

1.  **Design Prompt:** Design a URL shortener, notification system, or similar.
2.  **Component Diagram:** Boxes for API, DB, cache, workers, broker.
3.  **Data Model:** Sketch tables/keys and critical queries.
4.  **Failure Plan:** Describe what happens when cache or broker is down.

## 📖 Task Explanations

System design connects language skills to real architecture. Be explicit about bottlenecks and failure modes.

## 💡 What You'll Gain

A solid system design write-up tailored to Go service building blocks.

## 📚 Dictionary for Developers

*   **System Design:** Planning components, data, and trade-offs for a software system.
*   **Bottleneck Analysis:** Identifying the limiting part of a design.
*   **Single Point of Failure:** A component whose outage takes down the system.
