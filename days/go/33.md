# Day 33: Structured Logging

## 📝 Today's Tasks

1.  **slog Basics:** Use `log/slog` for structured logs.
2.  **Levels:** Log at info/warn/error appropriately.
3.  **Request IDs:** Add a request id field in middleware.
4.  **Avoid Noise:** Don't log secrets or huge payloads.

## 📖 Task Explanations

Structured logs unlock searchable operations data. Correlate requests with IDs. Prefer slog (stdlib) or a proven library; consistency matters more than the brand.

## 💡 What You'll Gain

Production-friendly structured logging with request correlation.

## 📚 Dictionary for Developers

*   **Structured Logging:** Logging as key-value fields rather than free-form strings.
*   **`log/slog`:** Go's standard structured logging package.
*   **Correlation ID:** An ID that ties logs for one request together.
