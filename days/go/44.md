# Day 44: Rate Limiting and Abuse Protection

## 📝 Today's Tasks

1.  **Why Rate Limit:** Discuss abuse, cost, and fairness.
2.  **Token Bucket / Fixed Window:** Implement a simple in-memory limiter middleware.
3.  **Response Headers:** Return Retry-After or rate-limit headers when useful.
4.  **Next Steps:** Note Redis-backed limiters for multi-instance deploys.

## 📖 Task Explanations

Rate limiting protects services and downstream dependencies. Start simple; graduate to distributed stores in production.

## 💡 What You'll Gain

Basic rate limiting middleware and an awareness of distributed needs.

## 📚 Dictionary for Developers

*   **Rate Limit:** A cap on how many requests a client may make in a period.
*   **Token Bucket:** An algorithm that refills tokens for allowing requests.
*   **Retry-After:** A header telling clients when to try again.
