# Day 17: Channels

## 📝 Today's Tasks

1.  **Unbuffered Channels:** Send/receive on unbuffered channels and observe synchronization.
2.  **Buffered Channels:** Use buffered channels for limited decoupling.
3.  **Close and Range:** Close a channel and range until closed.
4.  **Ownership:** Document which side closes the channel (sender owns close).

## 📖 Task Explanations

Channels communicate and synchronize goroutines. Unbuffered channels rendezvous; buffered channels add capacity. Only the sender should close; ranging stops when closed. Clear ownership prevents panics and deadlocks.

## 💡 What You'll Gain

Correct channel create/send/receive/close patterns and an intuition for buffering.

## 📚 Dictionary for Developers

*   **Channel:** A typed conduit for sending and receiving values between goroutines.
*   **Unbuffered Channel:** A channel with capacity 0 that synchronizes sender and receiver.
*   **Close:** Marking a channel so receivers eventually observe completion.
