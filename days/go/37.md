# Day 37: Queries, Scanning, and Repositories

## 📝 Today's Tasks

1.  **Scan Rows:** Scan into structs carefully; handle `sql.ErrNoRows`.
2.  **Repository Pattern:** Hide SQL behind a small repository interface.
3.  **Context Queries:** Use `QueryContext` / `ExecContext`.
4.  **SQL Injection:** Use parameterized queries only—never string-concat user input.

## 📖 Task Explanations

Repositories keep SQL out of handlers. Parameterized queries prevent injection. Context-aware queries honor cancellation.

## 💡 What You'll Gain

Clean data access with repositories, scanning, and safe parameters.

## 📚 Dictionary for Developers

*   **Repository:** A component that encapsulates persistence for a domain entity.
*   **Parameterized Query:** A query with placeholders bound separately from SQL text.
*   **`ErrNoRows`:** Error returned when a query expecting one row finds none.
