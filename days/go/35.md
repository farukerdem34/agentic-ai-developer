# Day 35: Context, Config & Logging Practice Project

## 📝 Today's Tasks

1.  **Project Goal:** Upgrade your REST API with config, context timeouts, and slog.
2.  **Timeouts:** Apply timeouts to handlers or outbound calls.
3.  **Health Endpoint:** Add `/healthz` that checks basic readiness.
4.  **Hardening:** Validate input strictly and log structured errors.

## 📖 Task Explanations

Operational readiness is part of API quality. Config + timeouts + logging turn a toy API into something you could deploy.

## 💡 What You'll Gain

A hardened API with configuration, observability basics, and disciplined validation.

## 📚 Dictionary for Developers

*   **Readiness:** Whether a service is ready to accept traffic.
*   **Health Check:** An endpoint used by orchestrators to probe service status.
