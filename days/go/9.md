# Day 9: Pointers, Values, and Memory Basics

## 📝 Today's Tasks

1.  **Pointer Basics:** Create pointers with `&` and dereference with `*`.
2.  **Nil Pointers:** Handle nil pointers safely; avoid panics.
3.  **Passing Arguments:** Compare pass-by-value for structs vs pointers in function calls.
4.  **Escape Analysis Intro:** Build with `-gcflags=-m` and observe a simple escape decision.

## 📖 Task Explanations

Everything in Go is passed by value, including pointers. Understanding when data lives on the stack vs heap (escape analysis) helps write efficient code, but clarity comes first. Nil checks and pointer discipline prevent common runtime panics.

## 💡 What You'll Gain

You will use pointers confidently, avoid nil panics, and have a basic sense of when values escape to the heap.

## 📚 Dictionary for Developers

*   **Pointer:** A value that stores the address of another value.
*   **Nil:** The zero value for pointers, interfaces, maps, slices, channels, and functions.
*   **Escape Analysis:** Compiler analysis deciding whether a value can live on the stack or must allocate on the heap.
*   **Pass by Value:** Function arguments receive copies of the caller's values.
