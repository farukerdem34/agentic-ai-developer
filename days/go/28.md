# Day 28: JSON APIs

## 📝 Today's Tasks

1.  **Encode JSON:** Respond with `encoding/json` and correct Content-Type.
2.  **Decode JSON:** Decode request bodies into structs with limits on body size.
3.  **Validation Basics:** Reject empty required fields with 400 responses.
4.  **Error JSON:** Return a consistent JSON error shape.

## 📖 Task Explanations

Most Go APIs speak JSON. Always bound decode sizes, validate input, and keep error envelopes consistent for clients.

## 💡 What You'll Gain

You will build JSON request/response endpoints with basic validation and consistent errors.

## 📚 Dictionary for Developers

*   **Content-Type:** A header declaring the media type of the body (e.g., application/json).
*   **DTO:** A data transfer object representing API payloads.
*   **Idempotent Design:** Designing endpoints so retries behave safely where required.
