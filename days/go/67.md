# Day 67: Distributed Tracing Basics

## 📝 Today's Tasks

1.  **Trace Concepts:** Spans, trace IDs, parent/child relationships.
2.  **OpenTelemetry Go:** Instrument an HTTP handler with a basic span.
3.  **Propagate Context:** Pass trace context across an outbound call.
4.  **Correlate Logs:** Include trace ids in logs when available.

## 📖 Task Explanations

Tracing explains latency across services. Start small—one service with clear spans—before building a mesh of instrumentation.

## 💡 What You'll Gain

Basic OpenTelemetry traces correlated with logs.

## 📚 Dictionary for Developers

*   **Span:** A unit of work within a trace.
*   **OpenTelemetry:** A vendor-neutral observability framework.
*   **Context Propagation:** Passing trace identifiers across process boundaries.
