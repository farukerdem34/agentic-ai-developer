# Day 53: Design Patterns That Fit Go

## 📝 Today's Tasks

1.  **Functional Options:** Implement config with functional options.
2.  **Decorators via Middleware:** Compose behavior without inheritance.
3.  **Strategy via Interfaces:** Swap algorithms behind an interface.
4.  **Avoid Pattern Theater:** Prefer straightforward code when patterns don't pay rent.

## 📖 Task Explanations

Classic OOP patterns map differently in Go. Interfaces, functions, and composition replace deep hierarchies. Learn patterns that match Go's grain.

## 💡 What You'll Gain

Practical Go-friendly patterns applied where they clarify design.

## 📚 Dictionary for Developers

*   **Functional Options:** A pattern for optional configuration via functions.
*   **Strategy Pattern:** Selecting an algorithm via an interface.
*   **Composition:** Building behavior by combining smaller parts.
