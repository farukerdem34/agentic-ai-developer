# Day 51: Project Layout and Modules

## 📝 Today's Tasks

1.  **Layout Options:** Study common layouts (e.g., `/cmd`, `/internal`, `/pkg`).
2.  **internal/ Visibility:** Use `internal` to prevent external imports.
3.  **cmd Entrypoints:** Separate binaries under `cmd/`.
4.  **Refactor:** Reorganize your project to match a clear layout.

## 📖 Task Explanations

Good layout scales with team size. `/internal` enforces boundaries. `/cmd` keeps mains thin.

## 💡 What You'll Gain

A professional Go repository layout that protects internals.

## 📚 Dictionary for Developers

*   **internal/:** A Go convention preventing other modules from importing the package.
*   **cmd/:** Directory for main packages / binaries.
*   **pkg/:** Optional home for publicly importable libraries.
