# Day 12: Maps

## 📝 Today's Tasks

1.  **Create Maps:** Initialize with `make` and literals; read, write, and delete keys.
2.  **Comma-ok Idiom:** Distinguish missing keys from zero values.
3.  **Iterate Maps:** Range over maps and note iteration order is randomized.
4.  **Reference Type:** Pass maps to functions and observe shared mutation.

## 📖 Task Explanations

Maps provide unordered key-value storage. Always use the comma-ok idiom when absence matters. Maps are reference types—be careful with concurrent access (covered later with sync).

## 💡 What You'll Gain

Idiomatic map usage, safe lookups, and awareness of iteration and sharing semantics.

## 📚 Dictionary for Developers

*   **Comma-ok Idiom:** The `value, ok := m[key]` pattern that reports whether a key exists.
*   **Map:** An unordered collection of key-value pairs.
*   **Reference Type:** A type whose header is copied, but refers to shared underlying data.
