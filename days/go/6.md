# Day 6: Structs and Composition

## 📝 Today's Tasks

1.  **Define Structs:** Model a domain type (e.g., `User` or `Product`) with fields.
2.  **Literal and Pointer Values:** Create values and pointers to structs.
3.  **Embedding:** Compose types using embedding (anonymous fields).
4.  **JSON Tags:** Add `json` struct tags and marshal/unmarshal a struct.

## 📖 Task Explanations

Structs are Go's primary way to define data shapes. Composition via embedding replaces classical inheritance. Struct tags annotate fields for encoding libraries. Getting comfortable with value vs pointer receivers and composition is essential for clean domain models.

## 💡 What You'll Gain

You will model data with structs, use embedding for composition, and serialize structs with JSON tags.

## 📚 Dictionary for Developers

*   **Struct:** A composite type that groups named fields.
*   **Embedding:** Including another type anonymously so its fields/methods are promoted.
*   **Struct Tag:** A string literal on a field used by reflection-based packages (e.g., `json`).
*   **Marshal:** Converting a Go value into an encoded format such as JSON.
