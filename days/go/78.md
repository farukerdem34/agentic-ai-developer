# Day 78: Outbox Pattern

## 📝 Today's Tasks

1.  **Dual Write Problem:** Explain why DB write + broker publish can diverge.
2.  **Transactional Outbox:** Write events to an outbox table in the same transaction.
3.  **Publisher Relay:** Build a relay that publishes pending outbox rows.
4.  **Idempotency Keys:** Ensure downstream dedupe.

## 📖 Task Explanations

The outbox pattern reliably publishes domain events without distributed transactions. It's a staple of serious event-driven systems.

## 💡 What You'll Gain

A transactional outbox with a simple publisher relay.

## 📚 Dictionary for Developers

*   **Dual Write:** Updating two systems without a shared transaction.
*   **Outbox:** A table storing events to publish reliably.
*   **Relay / Poller:** A process that publishes pending outbox events.
