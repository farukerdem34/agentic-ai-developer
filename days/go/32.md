# Day 32: Configuration Management

## 📝 Today's Tasks

1.  **Env-based Config:** Load settings from environment variables.
2.  **Defaults and Validation:** Apply defaults and fail fast on invalid config.
3.  **Config Struct:** Centralize config in a typed struct.
4.  **12-Factor Basics:** Keep secrets out of source control.

## 📖 Task Explanations

Configurable apps deploy cleanly across environments. Fail fast on bad config at startup. Environment variables remain the simplest starting point; files and flags can layer on later.

## 💡 What You'll Gain

Typed, validated configuration loaded from the environment.

## 📚 Dictionary for Developers

*   **12-Factor App:** A methodology for building SaaS apps, including strict config separation.
*   **Fail Fast:** Detecting and aborting on invalid configuration at startup.
*   **Secret:** Sensitive credential that must not be committed to source control.
