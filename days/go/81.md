# Day 81: WebSockets

## 📝 Today's Tasks

1.  **Upgrade Connection:** Implement a WebSocket endpoint (e.g., gorilla/websocket or nhooyr).
2.  **Broadcast Demo:** Fan out messages to connected clients.
3.  **Ping/Pong:** Keep connections alive and detect disconnects.
4.  **Auth Note:** Authenticate before upgrade carefully.

## 📖 Task Explanations

WebSockets enable full-duplex realtime communication. Heartbeats and auth at upgrade time are critical.

## 💡 What You'll Gain

A working WebSocket endpoint with basic broadcast and keepalive.

## 📚 Dictionary for Developers

*   **WebSocket:** A protocol for full-duplex communication over a single TCP connection.
*   **Upgrade:** Switching an HTTP connection to the WebSocket protocol.
*   **Heartbeat:** Periodic ping/pong to detect dead connections.
