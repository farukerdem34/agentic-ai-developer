# Day 68: Health, Readiness, and Timeouts

## 📝 Today's Tasks

1.  **Liveness vs Readiness:** Implement distinct probes if appropriate.
2.  **Dependency Checks:** Optionally check DB connectivity in readiness.
3.  **Timeouts Everywhere:** Enforce server, client, and DB timeouts.
4.  **Fail Partial:** Return degraded status carefully—don't lie healthy.

## 📖 Task Explanations

Orchestrators kill or stop traffic based on probes. Timeouts bound failure. Be honest in health responses.

## 💡 What You'll Gain

Meaningful probes and end-to-end timeout discipline.

## 📚 Dictionary for Developers

*   **Liveness:** Whether the process should be restarted.
*   **Readiness:** Whether the process should receive traffic.
*   **Degraded Mode:** Serving with reduced functionality when dependencies fail.
