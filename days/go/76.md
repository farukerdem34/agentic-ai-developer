# Day 76: Redis Caching

## 📝 Today's Tasks

1.  **Connect:** Use a Redis client to set/get values.
2.  **Cache-Aside:** Cache read-heavy responses with TTL.
3.  **Invalidation:** Invalidate or version keys on writes.
4.  **Failure Mode:** Degrade gracefully if Redis is down.

## 📖 Task Explanations

Caching reduces load and latency but introduces consistency challenges. TTL + invalidation strategy matters.

## 💡 What You'll Gain

Cache-aside reads with TTL and sane failure behavior.

## 📚 Dictionary for Developers

*   **Cache-Aside:** Application code loads DB then populates cache.
*   **TTL:** Time to live—automatic expiry for cached entries.
*   **Cache Stampede:** Many clients refreshing the same key simultaneously.
