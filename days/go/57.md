# Day 57: gRPC Services

## 📝 Today's Tasks

1.  **Implement Server:** Implement a gRPC service in Go.
2.  **Client Call:** Write a client that calls your service.
3.  **Errors:** Map domain errors to gRPC status codes.
4.  **HTTP Gateway Optional:** Note grpc-gateway if REST + gRPC needed.

## 📖 Task Explanations

gRPC is excellent for service-to-service calls with strict contracts. Status codes and metadata matter as much as payloads.

## 💡 What You'll Gain

A working gRPC server and client with proper status mapping.

## 📚 Dictionary for Developers

*   **gRPC:** A high-performance RPC framework built on HTTP/2 and protobufs.
*   **Stub:** Generated client/server code for an RPC service.
*   **Status Code:** A gRPC canonical error code such as NotFound or InvalidArgument.
