# Day 62: Worker Pools

## 📝 Today's Tasks

1.  **Fixed Workers:** Process jobs with a fixed number of workers.
2.  **Job/Result Channels:** Separate job submission from results.
3.  **Bounded Queues:** Cap queue size to protect memory.
4.  **Error Aggregation:** Collect per-job errors without losing successes.

## 📖 Task Explanations

Worker pools limit concurrency to protect databases and APIs. Bounded queues prevent unbounded memory growth.

## 💡 What You'll Gain

A practical worker pool with bounded buffering and error handling.

## 📚 Dictionary for Developers

*   **Worker Pool:** A fixed set of goroutines processing a job queue.
*   **Bounded Queue:** A queue with maximum capacity.
*   **Concurrency Limit:** A deliberate cap on parallel work.
