# Day 30: HTTP & REST Practice Project

## 📝 Today's Tasks

1.  **Project Goal:** Build a small in-memory REST API (e.g., notes or todos).
2.  **CRUD Routes:** Implement create, list, get, update, delete.
3.  **Middleware:** Add logging and recovery.
4.  **Graceful Shutdown:** Shut down with `http.Server.Shutdown` on signal.

## 📖 Task Explanations

Consolidate HTTP skills into a coherent CRUD API. Graceful shutdown is a professional touch that avoids dropping in-flight requests during deploys.

## 💡 What You'll Gain

A working REST-ish service with middleware and graceful shutdown—ready for databases next.

## 📚 Dictionary for Developers

*   **Graceful Shutdown:** Stopping a server after finishing in-flight requests.
*   **CRUD:** Create, Read, Update, Delete operations.
*   **In-Memory Store:** Temporary storage in process memory for learning and tests.
