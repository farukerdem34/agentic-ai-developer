# Day 47: JWT Authentication

## 📝 Today's Tasks

1.  **Issue Tokens:** Create JWTs after successful login.
2.  **Validate Tokens:** Middleware that verifies signature and expiry.
3.  **Claims:** Include subject and needed claims sparingly.
4.  **Https Note:** Discuss transport security requirements for tokens.

## 📖 Task Explanations

JWTs are common for stateless API auth. Keep secrets safe, expirations short, and validation strict. Prefer established libraries.

## 💡 What You'll Gain

Login that issues JWTs and middleware that protects routes.

## 📚 Dictionary for Developers

*   **JWT:** JSON Web Token—a signed token format often used for API auth.
*   **Claim:** A piece of information embedded in a token.
*   **Bearer Token:** A token sent in the Authorization header.
