# Day 87: Memory Profiling and Allocations

## 📝 Today's Tasks

1.  **Heap Profile:** Capture and inspect a heap profile.
2.  **Benchmark Allocs:** Use `-benchmem` to track allocations.
3.  **Reduce Allocs:** Preallocate slices, reuse buffers carefully, watch string concat.
4.  **GC Awareness:** Understand that less allocation reduces GC pressure.

## 📖 Task Explanations

Allocation rate often dominates latency at scale. Optimize the measured hot path—not everything.

## 💡 What You'll Gain

Ability to find and reduce costly allocations with evidence.

## 📚 Dictionary for Developers

*   **Heap Profile:** A profile of memory allocations still in use.
*   **GC Pressure:** Load on the garbage collector from high allocation rates.
*   **Buffer Reuse:** Reusing memory (e.g., sync.Pool) to cut allocations.
