# Day 41: Choosing a Router/Framework

## 📝 Today's Tasks

1.  **Survey Options:** Compare stdlib ServeMux vs chi vs echo vs gin at a high level.
2.  **Pick One:** Adopt one router for the next days (chi is a common stdlib-friendly choice).
3.  **Mount Routes:** Re-implement a few endpoints with the chosen router.
4.  **Stay Close to Stdlib:** Prefer frameworks that feel like `net/http`.

## 📖 Task Explanations

Frameworks speed routing and middleware, but stdlib knowledge remains transferable. Prefer thin frameworks that don't hide HTTP.

## 💡 What You'll Gain

A deliberate router choice and migrated routes without losing stdlib fluency.

## 📚 Dictionary for Developers

*   **Router:** A library that maps HTTP method+path to handlers.
*   **Chi:** A lightweight, idiomatic router compatible with `net/http`.
*   **Gin / Echo:** Popular fuller-featured Go web frameworks.
