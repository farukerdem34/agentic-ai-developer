# Day 88: Load Testing

## 📝 Today's Tasks

1.  **Pick a Tool:** Use vegeta, k6, or hey to load an endpoint.
2.  **Define SLOs:** Set target latency/error rates for the test.
3.  **Run Scenarios:** Baseline then stressed concurrency.
4.  **Interpret:** Distinguish app limits from machine limits.

## 📖 Task Explanations

Load tests validate assumptions. Define success criteria first, then increase load intentionally.

## 💡 What You'll Gain

A repeatable load test with documented results against an SLO-like target.

## 📚 Dictionary for Developers

*   **Load Test:** A test that applies concurrent traffic to measure performance.
*   **SLO:** Service Level Objective—a reliability/performance target.
*   **Saturation:** When a resource is fully utilized and queues grow.
