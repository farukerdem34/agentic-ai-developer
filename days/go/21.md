# Day 21: Unit Testing with go test

## 📝 Today's Tasks

1.  **Write Tests:** Create `*_test.go` with `TestXxx(t *testing.T)` functions.
2.  **Assertions Style:** Prefer clear `if` checks with `t.Fatalf` / `t.Errorf`.
3.  **Run Tests:** Use `go test`, `go test -v`, and package selection.
4.  **Helpers:** Use `t.Helper()` in test helper functions.

## 📖 Task Explanations

Testing is first-class in Go. The standard library testing package keeps tests simple and fast. Clear failure messages matter as much as coverage.

## 💡 What You'll Gain

You will write and run package tests with idiomatic failure reporting.

## 📚 Dictionary for Developers

*   **`testing.T`:** The type passed to tests for logging and failing.
*   **Unit Test:** A test that verifies a small unit of behavior in isolation.
*   **`t.Helper`:** Marks a function as a test helper so failures point to the caller.
