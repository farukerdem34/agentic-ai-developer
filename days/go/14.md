# Day 14: Error Handling Patterns

## 📝 Today's Tasks

1.  **Sentinel Errors:** Define and compare sentinel errors with `errors.Is`.
2.  **Wrapped Errors:** Wrap with `%w` and unwrap with `errors.As` / `errors.Is`.
3.  **Custom Error Types:** Implement `Error()` on a struct for richer context.
4.  **Don't Panic:** Convert a panic-based sketch into explicit error returns.

## 📖 Task Explanations

Idiomatic Go uses explicit `error` values, not exceptions. Wrapping preserves causality; sentinel and typed errors enable callers to branch correctly. Panics are for truly exceptional programmer mistakes, not normal control flow.

## 💡 What You'll Gain

You will design clear error APIs using wrap/Is/As and avoid panic-driven control flow.

## 📚 Dictionary for Developers

*   **Sentinel Error:** A predefined error value used for comparison.
*   **Error Wrapping:** Embedding an underlying error while adding context (`fmt.Errorf("...: %w", err)`).
*   **`errors.Is` / `errors.As`:** Helpers to match wrapped sentinel or typed errors.
