# Day 89: Performance Optimization Techniques

## 📝 Today's Tasks

1.  **I/O Bound vs CPU Bound:** Match optimizations to bottleneck type.
2.  **Caching Revisit:** Cache expensive reads after measuring.
3.  **Concurrency Caps:** Prevent overload with limits.
4.  **Before/After:** Record metrics proving improvement.

## 📖 Task Explanations

Performance work is a loop: measure, change, verify. Prefer architectural fixes (caching, batching, limits) over micro-tweaks when appropriate.

## 💡 What You'll Gain

A disciplined optimization with before/after evidence.

## 📚 Dictionary for Developers

*   **Bottleneck:** The resource that limits system throughput.
*   **Batching:** Grouping work to amortize overhead.
*   **Amortize:** Spreading a cost across many operations.
