# Day 24: Fuzzing and Test Doubles

## 📝 Today's Tasks

1.  **Fuzz Tests:** Write a `FuzzXxx` test for a parsing/validation function.
2.  **Seed Corpus:** Add interesting seed inputs.
3.  **Interfaces for Mocks:** Depend on interfaces so tests can inject fakes.
4.  **Fake Implementation:** Write a simple fake without a mocking framework.

## 📖 Task Explanations

Fuzzing finds unexpected inputs. Prefer fakes over heavy mock frameworks—interfaces make this easy. Keep fakes tiny and behavior-focused.

## 💡 What You'll Gain

You will fuzz a parser and test with a handmade fake implementation.

## 📚 Dictionary for Developers

*   **Fuzzing:** Automatically generating inputs to find panics or property violations.
*   **Fake:** A lightweight test double with working but simplified behavior.
*   **Seed Corpus:** Initial inputs that guide the fuzzer.
