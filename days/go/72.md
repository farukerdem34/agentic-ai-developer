# Day 72: Compose and Local Dependencies

## 📝 Today's Tasks

1.  **docker-compose:** Run API + Postgres (+ Redis optional) together.
2.  **Healthchecks:** Wait for DB readiness before app start if needed.
3.  **Volumes:** Persist database data for local development.
4.  **Dev Workflow:** Document one-command local bring-up.

## 📖 Task Explanations

Compose reproduces production-like dependencies locally. Good DX means fewer 'works on my machine' surprises.

## 💡 What You'll Gain

A compose-based local environment for your service and database.

## 📚 Dictionary for Developers

*   **docker-compose:** A tool to define and run multi-container local stacks.
*   **Healthcheck:** A container-level check used for dependency ordering.
*   **Volume:** Persistent storage for container data.
