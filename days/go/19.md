# Day 19: Mutexes and Race Detector

## 📝 Today's Tasks

1.  **Shared State:** Protect a shared counter with `sync.Mutex`.
2.  **RWMutex:** Read-heavy scenario with `sync.RWMutex` when appropriate.
3.  **Race Detector:** Run tests/binaries with `-race` and fix a deliberate race.
4.  **Prefer Channels?:** Discuss when mutexes vs channels fit better.

## 📖 Task Explanations

When goroutines share memory, you must synchronize. The race detector is essential. Idiomatic Go often prefers communicating by channels, but mutexes are appropriate for protecting invariants in shared structures.

## 💡 What You'll Gain

You will protect shared state correctly and use the race detector as a standard tool.

## 📚 Dictionary for Developers

*   **Data Race:** Concurrent access to shared memory with at least one write, without sync.
*   **Mutex:** A lock allowing one goroutine at a time into a critical section.
*   **`-race`:** Build/test flag enabling Go's race detector.
