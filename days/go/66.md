# Day 66: Metrics

## 📝 Today's Tasks

1.  **What to Measure:** Latency, traffic, errors, saturation (Golden Signals).
2.  **Prometheus Client:** Expose a `/metrics` endpoint with key counters/histograms.
3.  **Label Discipline:** Avoid high-cardinality labels.
4.  **Dashboards Note:** Sketch what you'd graph.

## 📖 Task Explanations

Metrics tell you how the system behaves in production. Start with golden signals and restrain label cardinality.

## 💡 What You'll Gain

Prometheus-style metrics exposed from a Go service with careful labels.

## 📚 Dictionary for Developers

*   **Golden Signals:** Latency, traffic, errors, saturation—core SRE metrics.
*   **Histogram:** A metric capturing distribution of values such as latency.
*   **Cardinality:** The number of unique time series created by label combinations.
