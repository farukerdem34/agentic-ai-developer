# Day 79: Background Jobs and Workers

## 📝 Today's Tasks

1.  **Job Model:** Define job types and payloads.
2.  **Worker Process:** Run a separate worker binary from `cmd/`.
3.  **Retries/DLQ:** Retry failures and park poison messages.
4.  **Observability:** Log job ids and durations.

## 📖 Task Explanations

Background workers handle email, image processing, and other slow tasks. Isolate them from request path latency.

## 💡 What You'll Gain

A worker binary processing jobs with retries and clear logs.

## 📚 Dictionary for Developers

*   **DLQ:** Dead-letter queue for messages that repeatedly fail.
*   **Poison Message:** A message that consistently crashes or fails processing.
*   **Job Payload:** The data required to execute a background job.
