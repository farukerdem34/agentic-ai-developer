# Day 11: Slices and Arrays

## 📝 Today's Tasks

1.  **Arrays vs Slices:** Create arrays and slices; note fixed vs dynamic length.
2.  **Append and Capacity:** Use `append`, observe length vs capacity, and preallocate with `make`.
3.  **Slicing:** Practice sub-slices and understand aliasing (shared backing arrays).
4.  **Copy:** Use `copy` to avoid unintended shared mutations.

## 📖 Task Explanations

Slices are the workhorse collection in Go. They reference a backing array; misunderstanding capacity and aliasing causes subtle bugs. Prefer slices over arrays for almost all application code.

## 💡 What You'll Gain

Confident use of slices, append, capacity planning, and safe copying.

## 📚 Dictionary for Developers

*   **Slice:** A descriptor of a contiguous segment of an array (pointer, length, capacity).
*   **Capacity:** The size of the backing array available to a slice without reallocating.
*   **Backing Array:** The underlying array that stores a slice's elements.
