# Day 10: Structs & Interfaces Practice Project

## 📝 Today's Tasks

1.  **Project Goal:** Build a tiny library (e.g., shapes or payment methods) using structs + interfaces.
2.  **Multiple Implementations:** Provide at least two concrete types for one interface.
3.  **Polymorphic Call Sites:** Invoke behavior through the interface in `main`.
4.  **Tests Preview:** Write at least one `_test.go` that asserts interface behavior.

## 📖 Task Explanations

Consolidate structs, methods, and interfaces into a cohesive mini-library. Prefer small interfaces and clear package boundaries. This pattern is the backbone of testable Go services.

## 💡 What You'll Gain

Practical experience modeling behavior with interfaces and swapping implementations—foundation for larger systems.

## 📚 Dictionary for Developers

*   **Polymorphism:** Using different concrete types through a shared interface.
*   **Test File:** A file named `*_test.go` containing tests run by `go test`.
