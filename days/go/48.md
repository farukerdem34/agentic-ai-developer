# Day 48: Sessions, Cookies, and CSRF Basics

## 📝 Today's Tasks

1.  **Server Sessions:** Contrast session cookies vs JWT bearer tokens.
2.  **Cookie Flags:** Use HttpOnly, Secure, SameSite appropriately.
3.  **CSRF Awareness:** Explain CSRF risks for cookie-based auth.
4.  **Choose Deliberately:** Document which model your API uses and why.

## 📖 Task Explanations

Cookie sessions suit browsers; bearers suit many APIs/SPAs/mobile clients. Cookie flags and CSRF protections matter for browser apps.

## 💡 What You'll Gain

Clear understanding of session vs token auth and cookie security flags.

## 📚 Dictionary for Developers

*   **HttpOnly:** A cookie flag that blocks JavaScript access.
*   **SameSite:** A cookie attribute that mitigates some CSRF vectors.
*   **CSRF:** Cross-Site Request Forgery—tricking a browser into authenticated requests.
