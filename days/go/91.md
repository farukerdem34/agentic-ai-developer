# Day 91: Code Review Culture for Go

## 📝 Today's Tasks

1.  **Review Checklist:** Idiomatic naming, error handling, ctx propagation, tests.
2.  **Small PRs:** Practice splitting a change into reviewable chunks.
3.  **Comments Style:** Prefer questions and rationale over nitpicks.
4.  **Self-Review:** Review your own diff before requesting eyes.

## 📖 Task Explanations

Code review is how teams stay healthy. Go's simplicity helps reviews focus on correctness and clarity.

## 💡 What You'll Gain

A practical Go code-review checklist and better PR habits.

## 📚 Dictionary for Developers

*   **Code Review:** Peer examination of changes before merge.
*   **Nitpick:** A minor stylistic comment; keep them optional/lightweight.
*   **Self-Review:** Reading your own PR carefully before asking for review.
