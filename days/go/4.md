# Day 4: Control Flow and Functions

## 📝 Today's Tasks

1.  **Conditionals and Switches:** Write examples with `if`, `else`, and `switch` (including switch on true).
2.  **Loops:** Use `for` in its three common forms (C-style, while-style, range over a string).
3.  **Functions:** Write functions with multiple return values and named returns.
4.  **Defer:** Use `defer` to ensure a cleanup action runs when a function exits.

## 📖 Task Explanations

Go keeps control flow simple: one looping construct (`for`), expression-oriented `switch`, and functions that often return `(result, error)`. `defer` schedules cleanup and is a cornerstone of resource management. Mastering these patterns makes later error handling and concurrency code easier to read.

## 💡 What You'll Gain

You will write clear control flow, multi-return functions, and use `defer` for cleanup—core idioms used throughout Go codebases.

## 📚 Dictionary for Developers

*   **Multiple Return Values:** A Go feature where functions return more than one value, commonly `(T, error)`.
*   **`defer`:** A statement that schedules a function call to run when the surrounding function returns.
*   **`range`:** A form of `for` that iterates over arrays, slices, maps, strings, or channels.
*   **Named Returns:** Named result parameters that can improve readability for complex returns.
