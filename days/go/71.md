# Day 71: Dockerfile for Go

## 📝 Today's Tasks

1.  **Multi-stage Build:** Compile in a builder image; copy binary to a minimal runtime image.
2.  **Static Binary Tips:** Understand CGO_ENABLED=0 when appropriate.
3.  **Non-root User:** Run as non-root in the final image.
4.  **Build Locally:** `docker build` and run your API container.

## 📖 Task Explanations

Multi-stage Docker builds keep images small and secure. Prefer distroless/scratch/alpine thoughtfully; reproducibility beats novelty.

## 💡 What You'll Gain

A minimal Docker image for your Go service built via multi-stage Dockerfile.

## 📚 Dictionary for Developers

*   **Multi-stage Build:** A Dockerfile technique using multiple FROM stages.
*   **CGO:** cgo enables calling C code; it complicates static builds.
*   **Distroless:** Minimal images containing little beyond your app.
