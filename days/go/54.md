# Day 54: Wiring and Lifecycle

## 📝 Today's Tasks

1.  **Manual DI:** Construct dependencies in main and inject them.
2.  **Lifecycle Hooks:** Start/stop background workers cleanly.
3.  **Avoid Global State:** Prefer explicit dependencies.
4.  **Smoke Test:** Boot the app in tests with fake collaborators.

## 📖 Task Explanations

Go usually favors explicit wiring in `main` over heavy DI containers. Clear lifecycle management prevents leaked goroutines.

## 💡 What You'll Gain

Explicit dependency wiring and clean process lifecycle management.

## 📚 Dictionary for Developers

*   **DI:** Dependency Injection—providing dependencies from the outside.
*   **Composition Root:** The place (usually main) where the object graph is built.
*   **Lifecycle:** How components start, run, and shut down.
