# Day 52: Clean Architecture in Go

## 📝 Today's Tasks

1.  **Layers:** Separate handlers, use cases/services, and repositories.
2.  **Dependencies Point Inward:** Domain should not import HTTP or SQL packages.
3.  **Interfaces at Boundaries:** Define interfaces where layers meet.
4.  **Refactor Vertical Slice:** Move one feature into clean layers.

## 📖 Task Explanations

Clean Architecture keeps business rules independent of frameworks and databases. In Go, keep it pragmatic—don't over-abstract.

## 💡 What You'll Gain

A feature structured with clear layers and inward dependencies.

## 📚 Dictionary for Developers

*   **Use Case / Interactor:** Application logic for a business action.
*   **Dependency Rule:** Source code dependencies point toward domain/policy.
*   **Pragmatic Clean Arch:** Applying boundaries without unnecessary ceremony.
