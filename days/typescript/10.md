# Day 10: Composite Types Practice Project

## 📝 Today's Tasks

1.  **Project Goal:** Model a small domain (tasks, cart, or feature flags) with objects, tuples, and literal unions.
2.  **State Transitions:** Write helpers that only allow valid status transitions.
3.  **Readonly API:** Export readonly collections and immutable-friendly update helpers.
4.  **Zero any:** Keep `strict` on; ban `any` and unchecked assertions.

## 📖 Task Explanations

Wire composite types into a coherent domain. Finite state unions + transition helpers showcase TypeScript's value beyond mere annotations.

## 💡 What You'll Gain

A typed domain model with literal-union states, safe transitions, and readonly boundaries.

## 📚 Dictionary for Developers

*   **Finite State Model:** A closed set of allowed states expressed in types.
*   **Transition Helper:** A function encoding legal state changes.
*   **Unchecked Assertion:** Forcing a type with `as` without proof—use sparingly.
