# Day 47: Path Aliases and Project References

## 📝 Today's Tasks

1.  **Paths Mapping:** Configure `baseUrl` + `paths` for clean imports.
2.  **Runtime Reality:** Ensure aliases work at runtime (bundler/tsconfig-paths/Node subpath).
3.  **Project References:** Split a solution into referenced TS projects.
4.  **Build Graph:** Use `tsc -b` for composite builds.

## 📖 Task Explanations

Aliases improve DX but need runtime support. Project references scale monorepos with clearer build boundaries—learn the mechanics before adopting heavyweight tools.

## 💡 What You'll Gain

Working path aliases and a basic project-references setup.

## 📚 Dictionary for Developers

*   **Path Alias:** A compile-time import shortcut mapped in tsconfig.
*   **Project Reference:** Linking TS projects into a build graph.
*   **Composite Project:** A TS project set up to be referenced (`composite: true`).
