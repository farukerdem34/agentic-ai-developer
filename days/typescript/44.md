# Day 44: Publishing Consumer-Facing Types

## 📝 Today's Tasks

1.  **types/typings Fields:** Configure package.json `types` (and exports) for declaration emit.
2.  **declaration Emit:** Enable `declaration` / `declarationMap` in tsconfig.
3.  **API Review:** Inspect generated `.d.ts` for accidental internal leaks.
4.  **SemVer Mentality:** Note that removing an exported type is a breaking change.

## 📖 Task Explanations

If you publish packages, your `.d.ts` is the product. Emit declarations, review them, and treat exported types as versioned API.

## 💡 What You'll Gain

A package setup that emits clean declaration files suitable for consumers.

## 📚 Dictionary for Developers

*   **Declaration Emit:** Compiler output of `.d.ts` files.
*   **`package.json` types:** Entry pointing consumers to your declarations.
*   **Breaking Type Change:** A type update that breaks downstream compiles.
