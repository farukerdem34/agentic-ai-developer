# Day 57: Mocking, Fakes, and Testing with Interfaces

## 📝 Today's Tasks

1.  **Interface-driven Fakes:** Build fakes implementing repository interfaces.
2.  **Mock Minimalism:** Prefer fakes over heavy mocking libraries when possible.
3.  **Async Tests:** Test promise success/failure paths cleanly.
4.  **Determinism:** Avoid flakiness (time, randomness) with injection.

## 📖 Task Explanations

Types make fakes easy: implement the interface and you're done. Deterministic tests depend on injecting clocks/randomness.

## 💡 What You'll Gain

Interface-based testing with fakes and deterministic async tests.

## 📚 Dictionary for Developers

*   **Fake:** A lightweight working stand-in collaborator.
*   **Mock:** A stand-in focused on interaction verification.
*   **Flaky Test:** A test that fails intermittently.
