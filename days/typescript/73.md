# Day 73: DTO Mapping and Layer Boundaries

## 📝 Today's Tasks

1.  **Define Layers:** Persistence ↔ Domain ↔ API DTOs.
2.  **Mappers:** Write pure mapper functions with explicit types.
3.  **Reject Leakage:** Ensure DB-only fields never reach public responses.
4.  **Test Mappers:** Unit test mapping edge cases.

## 📖 Task Explanations

Mappers are where types meet policy (what can leak). Keep them pure and tested. Coupled with schemas, they form a robust boundary layer.

## 💡 What You'll Gain

Clear DTO/domain mapping that prevents sensitive field leakage.

## 📚 Dictionary for Developers

*   **Mapper:** A pure function converting one shape to another.
*   **Layer Boundary:** The edge between architectural layers.
*   **Leakage:** Exposing internal fields across a boundary.
