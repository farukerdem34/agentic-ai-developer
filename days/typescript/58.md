# Day 58: Type-level Testing

## 📝 Today's Tasks

1.  **expectTypeOf / tsd / eslint type tests:** Pick an approach and write type tests.
2.  **Assert Assignability:** Prove that allowed values compile and disallowed values fail.
3.  **Utility Coverage:** Type-test your generic utilities from earlier days.
4.  **CI Include:** Run type tests in CI.

## 📖 Task Explanations

Type tests lock DX contracts—autocomplete shapes, rejected inputs, inferred outputs. Library authors treat them as first-class.

## 💡 What You'll Gain

Automated type tests protecting your utility/library APIs.

## 📚 Dictionary for Developers

*   **Type Test:** A test asserting compile-time type behavior.
*   **Assignability Assertion:** Checking whether type A is assignable to B.
*   **DX Contract:** The expected IDE/type behavior for consumers.
