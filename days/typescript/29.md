# Day 29: asserts, unknown, and never Discipline

## 📝 Today's Tasks

1.  **Asserts Signatures:** Write `asserts value is T` helpers for invariants.
2.  **unknown First:** Type external input as `unknown`, then guard/parse.
3.  **never Exhaustiveness:** Implement a `assertNever(x: never)` helper.
4.  **Limit `as`:** Replace two unsafe assertions with proper narrowing.

## 📖 Task Explanations

`asserts` narrows for the rest of a scope when a check throws on failure. Starting from `unknown` forces honest validation. Treat `as` as a last resort with a comment explaining why it's safe.

## 💡 What You'll Gain

Strong habits: unknown-in, narrow/assert, exhaust with never, minimize assertions.

## 📚 Dictionary for Developers

*   **Asserts Signature:** `asserts x is T`—throws if false; narrows afterward.
*   **Unsafe Assertion:** `as T` without sufficient proof.
*   **Invariant Check:** Runtime validation of a condition that must hold.
