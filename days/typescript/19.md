# Day 19: Abstract Classes vs Interfaces

## 📝 Today's Tasks

1.  **Abstract Class:** Share code + force subclasses to implement abstract methods.
2.  **Multiple implements:** Implement multiple interfaces on one class.
3.  **Decision Matrix:** Document when abstract class beats interface-only design.
4.  **Anti-pattern:** Avoid deep abstract hierarchies for simple data.

## 📖 Task Explanations

Abstract classes bundle shared implementation with contracts. Interfaces remain the lightest contract tool. Soft rule: start with interfaces; introduce abstract classes when shared behavior repeatedly duplicates.

## 💡 What You'll Gain

Practical judgment for abstract classes versus interfaces.

## 📚 Dictionary for Developers

*   **Abstract Method:** Declared without body; subclasses must implement.
*   **Shared Behavior:** Concrete methods usable by all subclasses.
*   **Over-abstraction:** Too many layers for the problem's size.
