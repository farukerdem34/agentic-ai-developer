# Day 2: Type Annotations and Type Inference

## 📝 Today's Tasks

1.  **Annotate Variables:** Declare variables with explicit `: string`, `: number`, and `: boolean` annotations.
2.  **Trust Inference:** Initialize values and let TypeScript infer types; hover in your IDE to confirm.
3.  **Annotate Boundaries:** Explicitly type function parameters and exported return types; leave obvious locals inferred.
4.  **Read Diagnostics:** Intentionally introduce type mismatches and practice reading the error messages carefully.

## 📖 Task Explanations

Annotations document intent; inference removes noise. Professional TypeScript annotates module boundaries (exports, public function params) and lets the compiler infer straightforward locals. Learning to read diagnostics is as important as writing types—errors point to real design issues.

## 💡 What You'll Gain

You will balance annotations vs inference, type public APIs explicitly, and use compiler errors as feedback instead of silencing them.

## 📚 Dictionary for Developers

*   **Type Annotation:** Explicitly declaring a value's type with `: Type`.
*   **Type Inference:** The compiler determining a type from initialization/usage.
*   **Module Boundary:** The exported surface other files depend on.
*   **Diagnostic:** A compiler error or warning explaining a type problem.
