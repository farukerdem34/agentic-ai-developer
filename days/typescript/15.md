# Day 15: Interfaces Practice Project

## 📝 Today's Tasks

1.  **Project Goal:** Design a typed data-access interface (repository) with fake and HTTP stub implementations.
2.  **Contract First:** Define request/response interfaces before implementations.
3.  **Swap Implementations:** Depend on the repository interface from a service layer.
4.  **Error Contract:** Define a shared error interface for failure cases.

## 📖 Task Explanations

Interface-first design enables testing and transport swaps. Keep contracts stable; let implementations vary. This is how NestJS/Next-style codebases stay testable.

## 💡 What You'll Gain

An interface-driven mini-architecture with two implementations and a shared error contract.

## 📚 Dictionary for Developers

*   **Repository Interface:** Abstraction over persistence/transport.
*   **Stub:** A minimal stand-in implementation.
*   **Stable Contract:** An API shape that changes rarely and carefully.
