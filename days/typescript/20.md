# Day 20: Classes Practice Project

## 📝 Today's Tasks

1.  **Project Goal:** Build a small domain (bank accounts, inventory items, or RPG entities) using classes.
2.  **Encapsulation:** Keep critical state private; mutate only via validated methods.
3.  **Polymorphism:** Process mixed subclasses through a shared interface/base type.
4.  **Reflect:** Write 5 lines on when you'd rewrite this with functions + interfaces instead.

## 📖 Task Explanations

Practice day for OOP-in-TS done carefully: encapsulation first, inheritance last. The reflection note locks in judgment, not dogma.

## 💡 What You'll Gain

A class-based domain with encapsulation, polymorphic use, and conscious design notes.

## 📚 Dictionary for Developers

*   **Domain Entity:** An object at the heart of business logic.
*   **Validation on Write:** Checking invariants when state changes.
*   **Design Judgment:** Choosing OOP vs FP styles based on context.
