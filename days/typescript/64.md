# Day 64: AbortSignal, Timeouts, and Cancellation Typing

## 📝 Today's Tasks

1.  **Accept AbortSignal:** Thread `signal?: AbortSignal` through async APIs.
2.  **Timeout Helper:** Create a timeout using AbortController.
3.  **Propagate Cancel:** Ensure cancellations abort fetch/work.
4.  **Type the Outcomes:** Distinguish abort errors from other failures.

## 📖 Task Explanations

Cancellation is part of async API design. Typing signals and abort outcomes prevents hung work and unclear failures.

## 💡 What You'll Gain

Cancellable async helpers with clear abort vs error modeling.

## 📚 Dictionary for Developers

*   **AbortSignal:** A signal object for communicating cancellation.
*   **AbortController:** Controller that creates and triggers an AbortSignal.
*   **Timeout:** Cancellation triggered after a time limit.
