# Day 42: Declaration Files and Ambient Types

## 📝 Today's Tasks

1.  **Write a .d.ts:** Create declarations for a small untyped JS module you invent.
2.  **declare Keywords:** Use `declare function` / `declare module` appropriately.
3.  **DefinitelyTyped Peek:** Inspect an `@types/*` package to see real-world `.d.ts` style.
4.  **Ship Types Mentality:** Treat `.d.ts` as a public product surface.

## 📖 Task Explanations

Declaration files describe JS so TypeScript can type-check calls. Reading DefinitelyTyped teaches idioms. Accurate declarations prevent lying types—worse than no types.

## 💡 What You'll Gain

Ability to author and read ambient declarations for JS interop.

## 📚 Dictionary for Developers

*   **Declaration File (`.d.ts`):** Type-only file describing existing JavaScript.
*   **Ambient Declaration:** Declaring values that exist at runtime without implementing them.
*   **DefinitelyTyped:** The community repo/packages providing `@types/*`.
