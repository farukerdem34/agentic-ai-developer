# Day 63: Typing Caught Errors and Error Hierarchies

## 📝 Today's Tasks

1.  **unknown in catch:** Use `catch (e: unknown)` and narrow safely.
2.  **Custom Error Classes:** Create domain errors with discriminants.
3.  **Type Guards for Errors:** Build `isDomainError` guards.
4.  **Never Swallow:** Ensure errors are logged/mapped, not ignored.

## 📖 Task Explanations

Modern TS types catch clauses as `unknown`. Narrowing + custom error classes create readable failure modes without `any`.

## 💡 What You'll Gain

Safe catch handling with typed domain errors.

## 📚 Dictionary for Developers

*   **Domain Error:** An error type representing a business failure.
*   **Error Narrowing:** Guards refining `unknown` into specific errors.
*   **Swallowed Error:** An error caught and ignored—usually a bug.
