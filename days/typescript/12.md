# Day 12: Type Aliases vs Interfaces — Decision Guide

## 📝 Today's Tasks

1.  **Recreate Shapes:** Express the same object as both `interface` and `type`.
2.  **Unions Need type:** Model unions/tuples only with type aliases.
3.  **Declaration Merging Demo:** Show interfaces merging; show types not merging.
4.  **Team Rule:** Write a short guideline: when to use which in your projects.

## 📖 Task Explanations

Both can describe objects. Type aliases uniquely cover unions, mapped types, and conditional types. Interfaces support declaration merging (useful—and dangerous—for ambient libs). Pick a convention and stay consistent.

## 💡 What You'll Gain

A deliberate rule set for `interface` vs `type` with merging awareness.

## 📚 Dictionary for Developers

*   **Type Alias:** `type Name = ...` naming any type.
*   **Declaration Merging:** Same-name interfaces combine declarations.
*   **Ambient Types:** Types describing values that exist outside your TS modules.
