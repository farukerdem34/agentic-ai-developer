# Day 16: Classes and Constructors in TypeScript

## 📝 Today's Tasks

1.  **Class Fields:** Declare typed fields and initialize them in the constructor.
2.  **Parameter Properties:** Use `constructor(private readonly id: string)` style.
3.  **Instance vs Static:** Add static factory methods (e.g., `User.create`).
4.  **Class Expressions:** Assign a class expression to a variable and use its type.

## 📖 Task Explanations

TypeScript adds types and visibility to JS classes. Parameter properties reduce boilerplate. Prefer factories/`readonly` when invariants matter. Classes are optional—many TS codebases prefer functions + interfaces—but you must read and write them fluently.

## 💡 What You'll Gain

Idiomatic typed classes with clear construction patterns.

## 📚 Dictionary for Developers

*   **Parameter Property:** Constructor param that also declares a field.
*   **Factory Method:** A static/function that creates validated instances.
*   **Invariant:** A rule that must hold for an object to be valid.
