# Day 18: Inheritance, override, and Polymorphism

## 📝 Today's Tasks

1.  **extends:** Build a small hierarchy (e.g., `Shape` → `Circle`).
2.  **override Keyword:** Mark overrides; enable `noImplicitOverride` if available.
3.  **Polymorphic Arrays:** Store subclasses in a base-typed collection and call shared methods.
4.  **Prefer Composition Note:** Refactor one inheritance use to composition + interfaces.

## 📖 Task Explanations

Inheritance is a tool, not a default. `override` and `noImplicitOverride` catch rename bugs. Prefer composition for flexible designs; keep inheritance shallow when used.

## 💡 What You'll Gain

Safe inheritance with explicit overrides and an appreciation for composition alternatives.

## 📚 Dictionary for Developers

*   **Inheritance:** Subclassing via `extends`.
*   **`override`:** Marks intentional method overrides.
*   **Composition:** Building behavior by combining smaller parts.
