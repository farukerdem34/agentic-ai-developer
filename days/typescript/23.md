# Day 23: Generic Interfaces, Classes, and Defaults

## 📝 Today's Tasks

1.  **Generic Interface:** Design `Repository<T>` or `ApiResponse<TData>`.
2.  **Generic Class:** Implement a simple `Box<T>` or in-memory store.
3.  **Default Type Params:** Provide defaults like `T = unknown` thoughtfully.
4.  **Nested Generics:** Compose `ApiResponse<Paginated<T>>`.

## 📖 Task Explanations

Generic interfaces/classes scale contracts across entity types. Defaults improve DX but can hide mistakes if too permissive—default to `unknown` more often than `any`.

## 💡 What You'll Gain

Reusable generic interfaces/classes with sensible defaults.

## 📚 Dictionary for Developers

*   **Generic Interface:** An interface parameterized by type variables.
*   **Default Type Parameter:** A fallback type arg used when omitted.
*   **Composition of Generics:** Nesting generic types for richer structures.
