# Day 34: Function Utility Types: ReturnType, Parameters, and Friends

## 📝 Today's Tasks

1.  **ReturnType:** Extract return types from existing functions.
2.  **Parameters / ConstructorParameters:** Type wrappers/decorators around existing functions.
3.  **Awaited:** Unwrap Promise types cleanly.
4.  **DRY Refactors:** Delete duplicated manual types replaceable by these utilities.

## 📖 Task Explanations

When you already have a well-typed function, utility types keep wrappers in sync automatically—excellent for middleware and adapters.

## 💡 What You'll Gain

Wrapper/adapters that stay synchronized via ReturnType/Parameters/Awaited.

## 📚 Dictionary for Developers

*   **`ReturnType<T>`:** The return type of a function type T.
*   **`Parameters<T>`:** Tuple of a function's parameter types.
*   **`Awaited<T>`:** Recursively unwraps Promise-like types.
