# Day 41: ES Modules in TypeScript

## 📝 Today's Tasks

1.  **Export/Import Styles:** Practice named exports, default exports, and `export type`.
2.  **type-only Imports:** Use `import type` to avoid runtime imports for types.
3.  **verbatimModuleSyntax Awareness:** Note modern flags that enforce type import clarity.
4.  **Circular Dependency Hunt:** Structure files to avoid cycles; fix one if it appears.

## 📖 Task Explanations

Modules are the packaging unit of TS apps. `import type` keeps type-only deps erasable and helps bundlers. Prefer named exports for APIs that grow.

## 💡 What You'll Gain

Clean module boundaries with correct type-only import habits.

## 📚 Dictionary for Developers

*   **`import type`:** Import used only in the type system.
*   **Named Export:** Exporting symbols by name.
*   **Circular Dependency:** Modules importing each other directly or indirectly.
