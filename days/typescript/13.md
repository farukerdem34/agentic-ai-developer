# Day 13: Structural Typing, extends, and implements

## 📝 Today's Tasks

1.  **Structural Assignability:** Assign objects with matching shapes without explicit implements.
2.  **extends Hierarchies:** Build interface hierarchies with `extends`.
3.  **Class implements:** Create a class that `implements` one or more interfaces.
4.  **Compatibility Cases:** Explore optional properties and excess property nuances.

## 📖 Task Explanations

TypeScript is structurally typed: if it has the required members, it matches. This is powerful for plain data and tricky with optional properties and function parameter variance (later). Use `implements` for documentation and guarantee—not because the language requires it for plain objects.

## 💡 What You'll Gain

Comfort with structural typing and intentional extends/implements usage.

## 📚 Dictionary for Developers

*   **Structural Typing:** Compatibility based on shape.
*   **Nominal Typing:** Compatibility based on explicit identity/brand.
*   **Assignability:** Whether a value of type A can be used where type B is expected.
