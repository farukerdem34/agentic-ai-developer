# Day 92: Monorepos and Shared Type Packages

## 📝 Today's Tasks

1.  **Workspace Layout:** Sketch or create a simple pnpm/npm workspaces layout.
2.  **Shared Types Package:** Put contracts in a package consumed by apps.
3.  **Versioning Options:** Discuss fixed vs independent versioning briefly.
4.  **Avoid Crossing Ends:** Keep server-only code out of shared packages.

## 📖 Task Explanations

Monorepos amplify TypeScript's strengths when shared contracts live in one package. Boundaries still matter.

## 💡 What You'll Gain

A basic workspace approach with a shared types package.

## 📚 Dictionary for Developers

*   **Workspace:** Multi-package repo managed together.
*   **Shared Types Package:** A package exporting contracts only.
*   **Independent Versioning:** Packages versioned separately.
