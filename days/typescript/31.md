# Day 31: Core Utility Types: Partial, Required, Readonly

## 📝 Today's Tasks

1.  **Partial Updates:** Type an update DTO with `Partial<T>`.
2.  **Required:** Use `Required<T>` for finalized objects after defaults apply.
3.  **Readonly Deep-ish:** Apply `Readonly<T>`; discuss shallow vs deep readonly.
4.  **Compose:** Combine utilities thoughtfully (e.g., `Readonly<Partial<T>>`).

## 📖 Task Explanations

Utility types remix existing types instead of duplicating shapes. Know they are shallow by default—deep variants need custom mapped types (later).

## 💡 What You'll Gain

Everyday fluency with Partial/Required/Readonly and awareness of shallowness.

## 📚 Dictionary for Developers

*   **Utility Type:** Built-in generic types transforming other types.
*   **DTO:** Data Transfer Object for inputs/outputs.
*   **Shallow Readonly:** Top-level fields readonly; nested objects still mutable.
