# Day 96: System Design with Types

## 📝 Today's Tasks

1.  **Design Prompt:** Design typed architecture for a URL shortener, notifications, or feature-flag service.
2.  **Contracts First:** Specify shared DTOs/events as TypeScript types.
3.  **Failure Modes:** Model errors as discriminated unions across services.
4.  **Writeup:** Produce a short design doc with diagrams and types.

## 📖 Task Explanations

System design for TS engineers starts with contracts and failure models. Types become the lingua franca across services.

## 💡 What You'll Gain

A design doc where TypeScript contracts and error unions are first-class.

## 📚 Dictionary for Developers

*   **Contract-first Design:** Defining typed interfaces before implementations.
*   **Cross-service Error Model:** Shared failure vocabulary across services.
*   **Design Doc:** A written plan of architecture and trade-offs.
