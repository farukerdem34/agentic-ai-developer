# Day 28: Discriminated Unions

## 📝 Today's Tasks

1.  **Tag Field:** Model variants with a shared discriminant like `type: 'success' | 'error'`.
2.  **Switch Exhaustiveness:** Switch on the tag; use `never` in the default for exhaustiveness.
3.  **Action Pattern:** Encode UI/domain events as discriminated unions.
4.  **Add a Variant:** Add a new variant and fix all non-exhaustive switches.

## 📖 Task Explanations

Discriminated unions are TypeScript's killer feature for state and events. A literal tag lets TS narrow each branch precisely. Exhaustiveness checks with `never` turn missed cases into compile errors—gold for refactors.

## 💡 What You'll Gain

Expert use of discriminated unions with exhaustive handling.

## 📚 Dictionary for Developers

*   **Discriminant:** A shared literal property identifying a union member.
*   **Exhaustiveness Check:** Ensuring every variant is handled.
*   **Tagged Union:** Another name for discriminated union.
