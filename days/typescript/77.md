# Day 77: Repository/Service Patterns in TypeScript

## 📝 Today's Tasks

1.  **Ports & Adapters Lite:** Define repository interfaces; implement in-memory + one other.
2.  **Services:** Put business rules in services depending on ports.
3.  **Keep Framework Out:** Ensure domain/services don't import framework types.
4.  **Test with Fakes:** Unit test services using in-memory repos.

## 📖 Task Explanations

Classic clean architecture, TS-flavored: interfaces as ports, pure-ish services, adapters at edges. Keep it pragmatic for small apps.

## 💡 What You'll Gain

A pragmatic ports-and-adapters slice with tested services.

## 📚 Dictionary for Developers

*   **Port:** An interface the domain depends on.
*   **Adapter:** A concrete implementation of a port.
*   **Application Service:** Orchestrates domain logic for a use case.
