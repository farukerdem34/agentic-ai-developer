# Day 6: Object Types and Property Modifiers

## 📝 Today's Tasks

1.  **Object Type Literals:** Define types for entities with required fields.
2.  **Optional Properties:** Use `prop?: T` and handle possibly-undefined access safely.
3.  **readonly Properties:** Prevent reassignment of fields that should be immutable after creation.
4.  **Excess Property Checks:** Pass object literals with typos and observe how TS catches them.

## 📖 Task Explanations

Object types describe shapes. Optional/readonly modifiers encode domain rules. Excess property checks apply especially to fresh literals—this catches bugs early but can surprise you when reusing variables (wider types).

## 💡 What You'll Gain

You will model objects with intentional optional/readonly fields and understand excess property checking behavior.

## 📚 Dictionary for Developers

*   **Object Type:** A type listing an object's properties and their types.
*   **Excess Property Check:** Rejects unknown properties on fresh object literals.
*   **Fresh Literal:** An object literal created at the call site rather than stored in a variable.
*   **Immutability (type-level):** Using `readonly` to prevent reassignment in the type system.
