# Day 11: Interfaces as Contracts

## 📝 Today's Tasks

1.  **Declare Interfaces:** Define `User`, `Product`, and `ApiError` interfaces with clear fields.
2.  **Satisfy Contracts:** Create plain objects that match interfaces.
3.  **Methods on Interfaces:** Add method signatures and implement them on objects/classes.
4.  **Small Interfaces:** Split a bloated interface into focused ones (read vs write, etc.).

## 📖 Task Explanations

Interfaces declare contracts. Keep them small and purposeful—fat interfaces force implementers to stub unused members. Prefer interfaces for object-shaped public APIs that may be extended.

## 💡 What You'll Gain

Clean interface contracts for entities and behaviors, split by responsibility.

## 📚 Dictionary for Developers

*   **Interface:** A TypeScript object contract declaration.
*   **Fat Interface:** An interface with too many unrelated members.
*   **Interface Segregation:** Prefer many small interfaces over one large one.
