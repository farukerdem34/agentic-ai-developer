# Day 7: Arrays, Tuples, and Readonly Collections

## 📝 Today's Tasks

1.  **Typed Arrays:** Work with `T[]` / `Array<T>` while preserving element types through map/filter.
2.  **Tuples:** Model fixed structures like `[latitude, longitude]` or `[statusCode, body]`.
3.  **Labeled Tuples:** Add labels to tuple elements for documentation.
4.  **Readonly Collections:** Expose `readonly T[]` from APIs to discourage mutation.

## 📖 Task Explanations

Arrays are variable-length homogeneous lists; tuples encode position-specific types. Prefer readonly collections at module boundaries to prevent accidental shared mutation—a common source of subtle bugs.

## 💡 What You'll Gain

Correct array vs tuple modeling and safer readonly collection APIs.

## 📚 Dictionary for Developers

*   **Tuple:** Fixed-length array type with per-position types.
*   **`ReadonlyArray<T>`:** Array type without mutating methods.
*   **Element Type Preservation:** Keeping generic element types through transformations.
*   **Labeled Tuple Element:** Named tuple slots for readability.
