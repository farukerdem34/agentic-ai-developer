# Day 97: Advanced Problem Solving & Debugging Types

## 📝 Today's Tasks

1.  **Reproduce a Type Puzzle:** Intentionally create a thorny error; solve it systematically.
2.  **Simplify:** Extract types, add intermediate aliases, use hover/inspect.
3.  **Tools:** Use `--traceResolution` or IDE type inspection when needed.
4.  **Postmortem:** Write what confused you and the rule you'll remember.

## 📖 Task Explanations

Expert TS devs debug types methodically: minimize, alias, inspect, fix. Puzzle speed comes from process.

## 💡 What You'll Gain

A disciplined type-debugging process documented via a short postmortem.

## 📚 Dictionary for Developers

*   **Type Puzzle:** A confusing type error requiring systematic debugging.
*   **Intermediate Alias:** A named type clarifying a complex expression.
*   **Trace Resolution:** Compiler tracing for module resolution issues.
