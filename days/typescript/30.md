# Day 30: Narrowing Practice Project

## 📝 Today's Tasks

1.  **Project Goal:** Build a typed message/event processor using discriminated unions.
2.  **Guards at Edges:** Parse unknown input with guards before processing.
3.  **Exhaustive Switch:** Fail compilation when a new event type is forgotten.
4.  **No any / minimal as:** Prefer guards and never helpers.

## 📖 Task Explanations

This project simulates real app event handling. The goal is zero mystery casts and compile-time breakages when events evolve.

## 💡 What You'll Gain

A production-minded event processor with guarded inputs and exhaustive unions.

## 📚 Dictionary for Developers

*   **Event Processor:** Code that routes/handles domain events.
*   **Edge Parsing:** Validating data as it enters your system.
*   **Refactor Safety:** Types that break builds when cases are missed.
