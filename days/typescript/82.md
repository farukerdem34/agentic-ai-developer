# Day 82: Function Overloads Done Right

## 📝 Today's Tasks

1.  **Declare Overloads:** Provide multiple call signatures + one implementation signature.
2.  **Prefer Unions When Possible:** Avoid overloads if a union/generic suffices.
3.  **Refine Returns:** Use overloads to couple input literals to return types.
4.  **Test Call Sites:** Ensure autocomplete picks the right overload.

## 📖 Task Explanations

Overloads are for relating specific inputs to specific outputs. Misused, they duplicate unions messily. Prefer simpler models first.

## 💡 What You'll Gain

Judicious overloads that improve call-site precision.

## 📚 Dictionary for Developers

*   **Function Overload:** Multiple call signatures for one function.
*   **Implementation Signature:** The broader signature used by the function body.
*   **Call-site Precision:** How accurately types reflect each call.
