# Day 86: satisfies and const Assertions

## 📝 Today's Tasks

1.  **as const:** Lock objects/arrays to literal types.
2.  **satisfies:** Check a value against a type while preserving literals.
3.  **Config Pattern:** Type a config with `satisfies` for safety + inference.
4.  **Replace Casts:** Remove a widening annotation that hurt literal inference.

## 📖 Task Explanations

`as const` and `satisfies` are modern precision tools. `satisfies` validates without widening—ideal for route maps, theme tokens, and config objects.

## 💡 What You'll Gain

Precise config/object typing using `as const` and `satisfies`.

## 📚 Dictionary for Developers

*   **`as const`:** Assertion producing the narrowest literal types.
*   **`satisfies`:** Validates against a type while keeping inferred literals.
*   **Widening:** Losing literal specificity to broader types.
