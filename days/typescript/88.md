# Day 88: Compile Performance and Incremental Builds

## 📝 Today's Tasks

1.  **incremental / composite:** Enable incremental builds and measure.
2.  **Skip Lib Check:** Know `skipLibCheck` trade-offs.
3.  **Find Slow Types:** Identify expensive types (huge unions) in your utilities.
4.  **Simplify Hot Types:** Replace one costly type with a simpler model.

## 📖 Task Explanations

Type performance is product performance for DX. Huge unions and recursive types can grind editors to a halt—measure and simplify.

## 💡 What You'll Gain

Faster incremental builds and at least one type-perf improvement.

## 📚 Dictionary for Developers

*   **Incremental Build:** Reusing previous compile info to speed builds.
*   **`skipLibCheck`:** Skips typechecking declaration files in node_modules.
*   **Type Perf:** How quickly the compiler/editor can check types.
