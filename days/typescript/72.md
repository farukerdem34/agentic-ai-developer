# Day 72: Branded Types and Opaque IDs

## 📝 Today's Tasks

1.  **Brand Pattern:** Create `UserId` / `OrderId` branded string types.
2.  **Prevent Mixing:** Show that branded IDs are not interchangeable.
3.  **Construct at Edges:** Only create brands after parsing/validation.
4.  **Serialize Carefully:** Strip brands at JSON boundaries intentionally.

## 📖 Task Explanations

Branded/opaque types fix primitive obsession (everything is a string). They cost little and prevent entire classes of bugs.

## 💡 What You'll Gain

Opaque ID types created only after successful parsing.

## 📚 Dictionary for Developers

*   **Branded Type:** A nominal-ish type created by intersecting with a unique brand.
*   **Opaque Type:** A type whose underlying representation is hidden.
*   **Primitive Obsession:** Overusing primitives where domain types are safer.
