# Day 43: Module Augmentation and Global Augmentation

## 📝 Today's Tasks

1.  **Augment a Module:** Add fields to a third-party module's types via augmentation.
2.  **Global Augmentation:** Extend `global` interfaces carefully (e.g., Window).
3.  **Keep Scope Tight:** Prefer module augmentation over globals.
4.  **Document Hacks:** Comment why augmentation was necessary and how to remove it later.

## 📖 Task Explanations

Augmentation is powerful and invasive. Use it to bridge missing types, not to paper over design issues. Always leave breadcrumbs for removal.

## 💡 What You'll Gain

Controlled augmentations with documentation and an exit plan.

## 📚 Dictionary for Developers

*   **Module Augmentation:** Adding to an existing module's type surface.
*   **Global Augmentation:** Extending global types.
*   **Type Lie:** Declarations that don't match runtime reality.
