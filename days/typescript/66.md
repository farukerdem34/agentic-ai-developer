# Day 66: DOM Typings and Event Handling

## 📝 Today's Tasks

1.  **lib.dom Usage:** Type button clicks and input events correctly.
2.  **Event Target Narrowing:** Narrow `event.currentTarget` appropriately.
3.  **Create Element Generics:** Use `document.createElement` with tag-specific return types.
4.  **Avoid any in UI:** Replace loose typings in a small UI script.

## 📖 Task Explanations

DOM typings are rich but easy to bypass. Prefer precise event types and element generics—frontend frameworks rely on these fundamentals.

## 💡 What You'll Gain

Accurate DOM event and element typing without `any`.

## 📚 Dictionary for Developers

*   **lib.dom:** TypeScript's DOM type library.
*   **currentTarget vs target:** Element handling the event vs originating element.
*   **Tag-specific Element Type:** e.g., `HTMLInputElement` vs `HTMLElement`.
