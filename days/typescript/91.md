# Day 91: Code Review for TypeScript

## 📝 Today's Tasks

1.  **Review Checklist:** any usage, missing awaits, weak boundaries, public export creep.
2.  **Suggest Types:** Practice proposing a discriminated union instead of booleans.
3.  **Nit Budget:** Separate style nits from type correctness issues.
4.  **Self-review:** Review one of your earlier PRs/diffs with the checklist.

## 📖 Task Explanations

TS reviews focus on contracts and safety. Good reviewers improve types without bike-shedding formatting (tooling handles style).

## 💡 What You'll Gain

A TypeScript-focused review checklist and better review habits.

## 📚 Dictionary for Developers

*   **Export Creep:** Accidentally exporting internals.
*   **Bike-shedding:** Debating trivial style instead of substance.
*   **Contract Review:** Reviewing types as API design.
