# Day 51: Strictness Flags Beyond strict

## 📝 Today's Tasks

1.  **noUncheckedIndexedAccess:** Enable and fix fallout from index access.
2.  **exactOptionalPropertyTypes:** Understand stricter optional property semantics.
3.  **noImplicitOverride / noPropertyAccessFromIndexSignature:** Toggle and evaluate.
4.  **Adopt Incrementally:** Plan enabling one extra flag on an existing codebase.

## 📖 Task Explanations

`strict` is necessary but not sufficient for maximal safety. Extra flags catch real bugs with some friction—adopt with a migration plan.

## 💡 What You'll Gain

A stricter config profile and an incremental adoption plan for advanced flags.

## 📚 Dictionary for Developers

*   **noUncheckedIndexedAccess:** Adds `undefined` to index access results.
*   **exactOptionalPropertyTypes:** Distinguishes missing vs explicitly undefined.
*   **Migration Plan:** Steps to enable a strict flag without a big-bang rewrite.
