# Day 53: Formatting, Editor DX, and Consistency

## 📝 Today's Tasks

1.  **Prettier:** Add Prettier and format the repo.
2.  **EditorConfig / VS Code settings:** Share baseline settings.
3.  **Organize Types:** Consistent naming (I-prefixes? usually no), file layout conventions.
4.  **Save-the-Diff:** Ensure format-on-save won't churn unrelated files.

## 📖 Task Explanations

Formatting wars waste review bandwidth. Automate style so reviews focus on types and behavior. Agree on naming and folder conventions early.

## 💡 What You'll Gain

Consistent formatting and editor DX shared across a team.

## 📚 Dictionary for Developers

*   **Prettier:** Opinionated code formatter.
*   **Format on Save:** Editor feature auto-formatting on file save.
*   **Review Signal:** How clearly a PR shows meaningful changes.
