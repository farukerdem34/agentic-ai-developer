# Day 8: Enums vs Literal Union Types

## 📝 Today's Tasks

1.  **String and Numeric Enums:** Create both; prefer string enums when enums are needed.
2.  **Literal Unions:** Model states as `'idle' | 'loading' | 'success' | 'error'`.
3.  **Trade-offs Writeup:** Note bundling, reverse mapping, and tree-shaking differences.
4.  **Migrate Example:** Convert one enum-based design to a literal union + const object pattern.

## 📖 Task Explanations

Enums are a TypeScript feature with runtime emit; many teams prefer literal unions plus `as const` objects for simpler JS output and better type precision. Understand enums, then choose deliberately.

## 💡 What You'll Gain

A clear preference matrix for enums vs literal unions in real projects.

## 📚 Dictionary for Developers

*   **Enum:** Named set of constants with TypeScript-specific runtime behavior.
*   **Literal Union:** A union of specific literal values.
*   **Const Object Pattern:** Using `as const` objects as an enum alternative.
*   **Tree-shaking:** Bundlers removing unused exported code.
