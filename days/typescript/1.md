# Day 1: Introduction to TypeScript and Toolchain

## 📝 Today's Tasks

1.  **Install TypeScript:** Install the TypeScript compiler as a project dependency (`npm i -D typescript`) and optionally explore the global install trade-offs.
2.  **Verify Versions:** Confirm Node.js LTS and `npx tsc --version`; note how local project TypeScript beats global installs for reproducibility.
3.  **Initialize Config:** Run `npx tsc --init` and skim key options (`target`, `module`, `rootDir`, `outDir`, `strict`).
4.  **Hello Compile Loop:** Write `src/hello.ts`, compile with `tsc`, and run the emitted JavaScript with Node.

## 📖 Task Explanations

TypeScript is a typed superset of JavaScript that compiles to plain JS. Types are erased at runtime—they guide the compiler and editors, not the Node/browser runtime. Today you establish a reproducible toolchain: local `typescript`, `tsconfig.json`, and the edit → type-check → emit → run loop used in every professional project.

## 💡 What You'll Gain

By the end of today, you will set up TypeScript locally, generate and understand a basic `tsconfig.json`, compile a small program, and explain type erasure. This is the foundation for every later day.

## 📚 Dictionary for Developers

*   **TypeScript (TS):** A statically typed language that compiles to JavaScript.
*   **`tsc`:** The TypeScript compiler that type-checks and emits JavaScript.
*   **`tsconfig.json`:** Project configuration controlling compiler options.
*   **Type Erasure:** Compile-time types are removed from emitted JavaScript.
*   **Reproducible Toolchain:** Pinning tooling in package.json so every developer and CI use the same compiler.
