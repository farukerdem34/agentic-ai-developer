# Day 61: Promise Types and Async Functions

## 📝 Today's Tasks

1.  **Annotate Async:** Write async functions and inspect inferred `Promise<T>` returns.
2.  **Parallel vs Sequential:** Type `Promise.all` / `allSettled` flows carefully.
3.  **Rejection Typing Reality:** Note that Promise rejections are not strongly typed by default.
4.  **Wrapper Helper:** Build a typed `sleep` / retry lite helper.

## 📖 Task Explanations

Async functions return Promises; inference usually shines. Rejections remain weakly typed in TS—design explicit error strategies rather than pretending throws are typed.

## 💡 What You'll Gain

Solid async typing plus awareness of untyped rejections.

## 📚 Dictionary for Developers

*   **Promise<T>:** Async result producing a T (or rejecting).
*   **allSettled:** Waits for all promises, capturing success/failure per item.
*   **Untyped Rejection:** Thrown/rejected values not captured in Promise's type params.
