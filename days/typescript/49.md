# Day 49: ESM vs CommonJS Interop

## 📝 Today's Tasks

1.  **Module Formats:** Build a tiny ESM package and a CJS consumer (or vice versa).
2.  **NodeNext Settings:** Experiment with `module`/`moduleResolution` NodeNext.
3.  **Extension Rules:** Practice `.js` extensions in TS ESM imports when required.
4.  **Interop Flags:** Understand `esModuleInterop` / `allowSyntheticDefaultImports`.

## 📖 Task Explanations

Module interop is the most common TS/Node footgun. Learn NodeNext semantics early; document your package's module format.

## 💡 What You'll Gain

Working knowledge of ESM/CJS interop in TypeScript projects.

## 📚 Dictionary for Developers

*   **ESM:** ECMAScript modules (`import`/`export`).
*   **CommonJS:** Node's classic `require`/`module.exports` system.
*   **Dual Package Hazard:** Problems when a package is loaded as both ESM and CJS.
