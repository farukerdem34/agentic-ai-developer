# Day 71: Parse Don't Validate with Zod (or Valibot)

## 📝 Today's Tasks

1.  **Schema First:** Define Zod schemas for a User/Product.
2.  **Infer Types:** Use `z.infer<typeof schema>` as the source of truth.
3.  **safeParse:** Handle success/failure without throws at boundaries.
4.  **Replace Guards:** Swap hand-written guards for schemas where appropriate.

## 📖 Task Explanations

Schemas give one source of truth for runtime and types. 'Parse, don't validate' means producing typed data via parsers rather than boolean checks that leave types unchanged.

## 💡 What You'll Gain

Schema-first boundaries with inferred TypeScript types.

## 📚 Dictionary for Developers

*   **Zod:** A TypeScript-first schema validation library.
*   **`z.infer`:** Extracts a static type from a schema.
*   **Parse don't validate:** Produce typed values via parsing rather than only checking.
