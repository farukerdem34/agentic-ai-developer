# Day 9: Union and Intersection Types in Depth

## 📝 Today's Tasks

1.  **Build Unions:** Model values that can be one of several shapes (`User | Guest`).
2.  **Build Intersections:** Combine types with `&` (e.g., `Timestamped & Identifiable`).
3.  **Practical Mixing:** Create APIs that accept unions and return intersections where meaningful.
4.  **Pitfall Hunt:** Force a case where intersecting incompatible types yields `never` and explain why.

## 📖 Task Explanations

Unions express alternatives; intersections express combination. Overusing intersections of open object types can produce surprising `never` conflicts. Prefer discriminated unions for variants (next block).

## 💡 What You'll Gain

Fluent, intentional modeling with `|` and `&` without accidental `never` traps.

## 📚 Dictionary for Developers

*   **Union Type:** `A | B`—value is one of the members.
*   **Intersection Type:** `A & B`—value must satisfy all members.
*   **Compatible Types:** Types that can be combined without collapsing to `never`.
*   **Variant:** One alternative in a union of shapes.
