# Day 17: Access Modifiers and Privacy Models

## 📝 Today's Tasks

1.  **TS private/protected/public:** Enforce encapsulation at compile time.
2.  **Native #private:** Contrast TypeScript `private` with runtime `#field` privacy.
3.  **readonly + private:** Model immutable identities and encapsulated mutable state.
4.  **API Minimization:** Expose the smallest public surface that still gets the job done.

## 📖 Task Explanations

TypeScript `private` is erased—clever JS can still access it. `#private` is runtime-enforced. Use TS privacy for most app code; use `#private` when runtime isolation matters. Small public surfaces are easier to maintain.

## 💡 What You'll Gain

Encapsulation literacy across compile-time and runtime privacy models.

## 📚 Dictionary for Developers

*   **Compile-time Privacy:** Restricted by the type checker only.
*   **Runtime Privacy:** Enforced by the JavaScript engine (`#private`).
*   **Public Surface:** Members other modules are allowed to use.
