# Day 32: Pick, Omit, and API Shape Surgery

## 📝 Today's Tasks

1.  **Pick:** Create focused views with `Pick<User, 'id' | 'email'>`.
2.  **Omit:** Strip sensitive fields with `Omit<User, 'passwordHash'>`.
3.  **Public vs Persistence Models:** Separate DB entities from public API types using Pick/Omit.
4.  **Danger Zone:** Avoid Pick/Omit on types you don't own without documenting coupling.

## 📖 Task Explanations

Pick/Omit create purpose-built shapes. They couple you to the source type—helpful for DRY, risky when the source changes unexpectedly. For public libraries, consider explicit types.

## 💡 What You'll Gain

Safe API shape derivation with Pick/Omit and intentional model separation.

## 📚 Dictionary for Developers

*   **Model Separation:** Different types for DB, domain, and API layers.
*   **Coupling:** When changing one type forces changes in derived types.
*   **Sensitive Field:** Data that must not leak across boundaries.
