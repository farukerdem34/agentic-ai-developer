# Day 68: Isomorphic TypeScript Patterns

## 📝 Today's Tasks

1.  **Shared Types Package:** Create types shared by client and server.
2.  **Avoid DOM in Shared:** Keep shared modules runtime-agnostic.
3.  **Conditional Imports Awareness:** Note pitfalls of isomorphic bundles.
4.  **API Contract File:** Define request/response types consumed by both sides.

## 📖 Task Explanations

Isomorphic/shared types prevent client/server drift. Keep shared modules free of DOM/Node-only imports.

## 💡 What You'll Gain

A shared contract package suitable for full-stack TypeScript systems.

## 📚 Dictionary for Developers

*   **Isomorphic Code:** Code that can run in multiple runtimes.
*   **Shared Contract:** Types used by both client and server.
*   **Drift:** When client and server disagree on shapes.
