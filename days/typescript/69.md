# Day 69: Typing Fetch and HTTP Boundaries

## 📝 Today's Tasks

1.  **fetch Wrappers:** Build a typed `apiGet<T>` that still needs runtime validation (preview Zod).
2.  **Response Shapes:** Model success/error HTTP envelopes.
3.  **Status Narrowing:** Handle non-OK responses explicitly.
4.  **Never Trust JSON:** Keep `response.json()` as unknown until parsed.

## 📖 Task Explanations

HTTP is a trust boundary. Generics on fetch without validation are aspirational types—pair with runtime parsing next block.

## 💡 What You'll Gain

Honest HTTP typing that treats JSON as unknown until validated.

## 📚 Dictionary for Developers

*   **Trust Boundary:** Where data enters from outside your process.
*   **Aspirational Type:** A compile-time type not enforced at runtime.
*   **HTTP Envelope:** Standard wrapper for API success/error payloads.
