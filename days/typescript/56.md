# Day 56: Runtime Testing with Vitest or Jest

## 📝 Today's Tasks

1.  **Pick a Runner:** Set up Vitest (recommended DX) or Jest with TS.
2.  **Unit Tests:** Test pure functions with table-driven style cases.
3.  **Watch Mode:** Use watch for a fast feedback loop.
4.  **Coverage Optional:** Generate coverage and interpret responsibly.

## 📖 Task Explanations

Runtime tests prove behavior types cannot (I/O, logic branches, regressions). Keep tests close to pure logic; type tests come next.

## 💡 What You'll Gain

A working TS test runner with meaningful unit tests.

## 📚 Dictionary for Developers

*   **Unit Test:** Test of a small unit of behavior.
*   **Table-driven Tests:** Shared test logic over many input/output rows.
*   **Watch Mode:** Re-runs tests on file changes.
