## Day 8

### Mini Spec: LLM Chat with Tool Calling

#### Problem

Users ask questions about current/future events (e.g., "Who won the World Cup 2026?"). The LLM's knowledge is outdated and cannot answer accurately. Tool calling (webfetch) allows the LLM to retrieve real-time information.

#### Users

- **Casual User:** Asks questions, expects quick, accurate answers from AI

#### User Stories

1. **As a** casual user, **I want to** get accurate, up-to-date information from AI **so that** I can answer my questions faster.

   - AC: Given I ask "What time is it?", when the LLM answers, then the response contains the current local time from check_date API and matches within 1 minute.
   - AC: Given I ask "What is the best movie in 2022?", when the LLM answers using imdb_tool, then the movie title and rating match the top-rated movie from imdb_tool for 2022.
   - AC: Given I ask "Who won the World Cup 2026?", when the LLM answers using webfetch, then the response contains the country that won the World Cup 2026 according to the live source.
   - AC: Given I ask "Who won the World Cup 2026?", when the LLM needs current information, then the LLM uses tool calling to fetch the answer instead of guessing.

#### Constraints

1. Available tools: `webfetch`, `check_date`, `imdb_tool`
2. Must use existing LLM API — no model changes
3. Response time: under 1 second

#### Out of Scope (v1)

- Multi-language support
- Custom tool creation by users
- Chat history persistence

#### Components

| Component | Responsibility |
|-----------|----------------|
| Chat UI | User input, displays messages, shows tool results |
| Tool Router | Decides which tool to call based on user query |
| LLM API | Generates responses, triggers tool calls when needed |
| Tool Executor | Makes direct HTTP API calls to external services (webfetch, imdb, check_date) |
| Message Store | Persists conversation history |

### Trade-off Note

| Decision | Trade-off |
|----------|-----------|
| LLM decides tool selection | ✅ Flexible — handles synonyms, context, ambiguity. ❌ Slower — extra LLM call adds latency. |
| **Alternative rejected:** Keyword matching | ✅ Faster — no extra call needed. ❌ Brittle — misses synonyms ("What time is it?" works, "Current time?" might not). |
