### Give Feedback

**Sample diff:**

```python
# Before
def get_user(id):
    return db.query(f"SELECT * FROM users WHERE id = {id}")

# After
def get_user(user_id: int) -> dict:
    query = "SELECT * FROM users WHERE id = %s"
    return db.execute(query, (user_id,))
```

**Kind comment:** Specifying what type of data function returns is significant usage for you and other developers.

**Critical comment:** The return system works well, unless user does not exist with specified user id. There is no user id control or return null type.

### Receive Feedback

**Review comment received:** "This code is inefficient. You're making 3 API calls when you could make 1."

**Professional response:**

> "Thanks for your feedback. We're using 3 API calls because of modularity, different APIs, and reliability. If performance is a concern, we could measure first. Would that work?"
