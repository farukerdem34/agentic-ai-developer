### Components

|Component|Responsibility|
|-|-|
|UI|Renders product pages, cart, checkout form|
|Cart API|Business logic — add/remove items, calculate totals|
|Auth Service|Identity verification, session management|
|Order DB|Order DB	Persists cart state, order history|
|Payment Adapter|Wraps Stripe SDK, handles charges & refunds|
|Notification Job|Sends order confirmation emails asynchronously|

Each component should do one thing well.

### Trade-off Note:

|Decision|Trade-off|
|Cart stored in DB|Cart persists across devices and sessions|
|Alternantive Rejected: Store cart only in browser localStorage|Zero server cost. Lost if user switches device or clears cache, poor UX for returning custermers|
