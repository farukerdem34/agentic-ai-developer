## Day 11

### Testing Levels

| Level | Definition | Example (LLM Tool Calling) |
|-------|------------|---------------------------|
| Unit Test | Tests a small unit of code in isolation | Test Tool Router function with mocked LLM response |
| Integration Test | Tests how components work together | Test Tool Router calling real imdb_tool API |
| E2E Test | Tests a full user flow through the system | User asks "best movie 2022" → LLM responds with correct title |

### Bug Life Cycle

| Step | Action |
|------|--------|
| 1 | Bug reported (user finds issue) |
| 2 | Triage (assess severity/priority) |
| 3 | Assigned to developer |
| 4 | Developer reproduces the bug |
| 5 | Developer fixes the bug |
| 6 | QA verifies the fix |
| 7 | Bug closed |

### Quality ≠ Only QA — Developer Responsibilities

| # | Responsibility |
|---|----------------|
| 1 | Write unit and integration tests |
| 2 | Run linter (no warnings) |
| 3 | Self-review code before requesting review |
| 4 | Review other developers' code |
| 5 | Document public APIs and functions |
